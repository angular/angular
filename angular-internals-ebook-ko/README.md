# Angular 내부 구조: 개발자의 여정

> **Angular 핵심 아키텍처를 이해하기 위한 인터랙티브 가이드**

"Angular 내부 구조"에 오신 것을 환영합니다. 이 책은 개발자 Alex가 TaskMaster라는 완전한 작업 관리 애플리케이션을 만들면서 Angular 프레임워크의 내부 구조를 깊이 탐구하는 여정을 따라갑니다.

## 📖 이 책에 대하여

전통적인 기술 문서와 달리, 이 책은 **스토리텔링**을 사용하여 복잡한 개념을 설명합니다. 각 챕터는 Alex가 마주치는 실제 문제를 제시한 후, Angular의 내부 구조를 깊이 파고들어 문제를 해결합니다. 이 책을 읽고 나면 다음을 이해하게 됩니다:

- 의존성 주입이 실제로 어떻게 작동하는지
- 변경 감지의 비밀
- 컴포넌트 생명주기 동안 무슨 일이 일어나는지
- 템플릿이 실행 가능한 코드로 변환되는 과정
- 렌더링 엔진의 내부 작동 방식
- Zone.js와 비동기 작업 추적
- Signals를 사용한 현대적 반응성
- 라우터의 네비게이션 메커니즘

## 🎯 이 책은 누구를 위한 것인가요?

- 프레임워크의 "왜"를 이해하고 싶은 **중급에서 고급 Angular 개발자**
- Angular에 기여하고자 하는 **프레임워크 기여자**
- 아키텍처 결정을 내려야 하는 **기술 리드**
- 작동 원리를 이해하는 것을 좋아하는 **호기심 많은 개발자**

#### [1장: 의존성 주입의 미스터리](chapters/01-dependency-injection.md)

Alex는 서비스가 주입되지 않는 당혹스러운 오류를 만납니다. 이를 통해 다음을 이해합니다:

- 프로바이더 해결 알고리즘

**코드 예제**: 계층적 DI를 사용한 플러그인 시스템 구축

#### [2장: 변경 감지의 수수께끼](chapters/02-change-detection.md)
*"버튼을 클릭했는데 UI가 업데이트되지 않아!"*

버튼 클릭이 UI 업데이트를 트리거하지 않습니다. Alex는 다음을 발견합니다:
- Angular가 언제 업데이트해야 하는지 아는 방법

- OnPush 전략의 비밀
- Zone.js 통합

**코드 예제**: 실시간 대시보드 최적화

---

#### [3장: 생명주기 연대기](chapters/03-component-lifecycle.md)
*"정확히 언제 데이터를 로드해야 할까?"*

Alex는 컴포넌트 데이터를 언제 초기화해야 하는지 고민합니다. 이를 통해 다음을 알게 됩니다:
- 8가지 생명주기 훅 상세 설명
- 훅 실행 순서
- ViewChild와 ContentChild 타이밍
- Signal 기반 입력과 effects

**코드 예제**: 복잡한 데이터 로딩 시나리오

## 🛠️ 전제 조건

이 책을 최대한 활용하려면 다음이 필요합니다:

- ✅ **기본 Angular 지식** - 컴포넌트, 서비스, 템플릿
- ✅ **TypeScript 숙련도** - 타입, 데코레이터, 제네릭
- ✅ **JavaScript 능숙함** - 클로저, 프로미스, async/await
- ✅ **개발 환경** - Node.js, Angular CLI

**선택사항이지만 도움이 됨:**
- RxJS 기초 이해
- 빌드 도구에 대한 친숙함
- 복잡한 애플리케이션 디버깅 경험
