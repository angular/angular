# EnvironmentProviders использован в неверном контексте

Эта ошибка возникает, когда `EnvironmentProviders` используются в контексте, принимающем только обычные провайдеры,
например в массиве `providers` компонента. Провайдеры среды предназначены для конфигурации уровня приложения и могут
использоваться только в инжекторах среды (таких как корневой инжектор, настроенный в `bootstrapApplication`, или
конфигурации маршрутов).

## Распространенные сценарии {#common-scenarios}

### Использование `provideHttpClient()` в провайдерах компонента {#using-providehttpclient-in-component-providers}

Функции вроде `provideHttpClient()` возвращают `EnvironmentProviders`, которые нельзя использовать на уровне
компонента:

```ts
@Component({
  providers: [
    provideHttpClient(), // ОШИБКА: EnvironmentProviders нельзя использовать здесь
  ],
})
export class UserProfile {}
```

### Использование `importProvidersFrom()` в провайдерах компонента {#using-importprovidersfrom-in-component-providers}

Функция `importProvidersFrom()` также возвращает `EnvironmentProviders`:

```ts
@Component({
  providers: [
    importProvidersFrom(SomeModule), // ОШИБКА: нельзя использовать в провайдерах компонента
  ],
})
export class DataView {}
```

## Отладка ошибки {#debugging-the-error}

Переместите провайдеры среды в подходящее место:

### Для провайдеров уровня приложения {#for-application-wide-providers}

Настройте провайдеры среды в `bootstrapApplication`:

```ts
bootstrapApplication(App, {
  providers: [provideHttpClient(), importProvidersFrom(SomeModule)],
});
```

### Для провайдеров конкретного маршрута {#for-route-specific-providers}

Используйте массив `providers` в конфигурации маршрутов:

```ts
const routes: Routes = [
  {
    path: 'admin',
    component: AdminView,
    providers: [provideHttpClient(withInterceptors([authInterceptor]))],
  },
];
```

### Для сервисов уровня компонента {#for-component-level-services}

Если вам нужны сервисы с областью видимости компонента, используйте обычные провайдеры вместо провайдеров среды:

```ts
@Component({
  providers: [
    UserClient, // Обычный провайдер — работает
    {provide: API_URL, useValue: '/api'}, // Провайдер значения — работает
  ],
})
export class UserProfile {}
```

Сообщение об ошибке указывает, какой именно провайдер вызвал проблему. Убедитесь, что все элементы в массиве
`providers` вашего компонента являются обычными провайдерами, а не провайдерами среды, возвращаемыми такими функциями,
как `provideHttpClient()`, `provideRouter()` или `importProvidersFrom()`.
