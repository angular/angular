# EnvironmentProviders in wrong context {#environmentproviders-in-wrong-context}

Эта ошибка возникает, когда `EnvironmentProviders` используются в контексте, который принимает только обычные провайдеры, например в массиве `providers` компонента. Провайдеры среды предназначены для конфигурации на уровне всего приложения и могут использоваться только в инжекторах среды (например, в корневом инжекторе, настроенном в `bootstrapApplication`, или в конфигурациях маршрутов).

## Типичные сценарии {#common-scenarios}

### Использование `provideHttpClient()` в провайдерах компонента {#using-providehttpclient-in-component-providers}

Функции вроде `provideHttpClient()` возвращают `EnvironmentProviders`, которые нельзя использовать на уровне компонента:

```ts
@Component({
  providers: [
    provideHttpClient(), // ОШИБКА: EnvironmentProviders нельзя использовать здесь
  ],
})
export class UserProfile {}
```

### Использование `importProvidersFrom()` в провайдерах компонента {#using-importprovidersfrom-in-component-providers}

Функция `importProvidersFrom()` также возвращает `EnvironmentProviders`:

```ts
@Component({
  providers: [
    importProvidersFrom(SomeModule), // ОШИБКА: нельзя использовать для провайдеров компонента
  ],
})
export class DataView {}
```

## Отладка ошибки {#debugging-the-error}

Переместите провайдеры среды в подходящее место:

### Для провайдеров уровня приложения {#for-application-wide-providers}

Настройте провайдеры среды в `bootstrapApplication`:

```ts
bootstrapApplication(App, {
  providers: [provideHttpClient(), importProvidersFrom(SomeModule)],
});
```

### Для провайдеров уровня маршрута {#for-route-specific-providers}

Используйте массив `providers` в конфигурациях маршрутов:

```ts
const routes: Routes = [
  {
    path: 'admin',
    component: AdminView,
    providers: [provideHttpClient(withInterceptors([authInterceptor]))],
  },
];
```

### Для сервисов уровня компонента {#for-component-level-services}

Если нужны сервисы, ограниченные компонентом, используйте обычные провайдеры вместо провайдеров среды:

```ts
@Component({
  providers: [
    UserClient, // Обычный провайдер — работает
    {provide: API_URL, useValue: '/api'}, // Value-провайдер — работает
  ],
})
export class UserProfile {}
```

Сообщение об ошибке указывает, какой именно провайдер вызвал проблему. Убедитесь, что все элементы в массиве `providers` вашего компонента являются обычными провайдерами, а не провайдерами среды, возвращаемыми функциями вроде `provideHttpClient()`, `provideRouter()` или `importProvidersFrom()`.
