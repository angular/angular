# Несоответствие узлов при гидратации {#hydration-node-mismatch}

Эта ошибка означает, что в процессе гидратации Angular ожидал структуру DOM, аналогичную той, что была отрендерена и аннотирована при серверном рендеринге. Однако на клиенте дерево DOM отличалось от того, что было получено на сервере.

Эта ошибка обычно возникает при прямом манипулировании DOM с помощью нативных браузерных API, изменяющих структуру DOM за пределами того, что создал Angular. Она также возникает при использовании `innerHTML` или `outerHTML` для задания HTML-содержимого, что аналогично изменяет структуру DOM за пределами того, что создал Angular. Чтобы устранить проблему, выполните рефакторинг компонента с переходом на нативные Angular API вместо нативных браузерных API. Если это невозможно, добавьте атрибут `ngSkipHydration` к хост-узлу компонента — это отключит гидратацию для данного компонента и его дочерних элементов. `ngSkipHydration` следует использовать лишь как крайнюю меру; его наличие следует рассматривать как ошибку, требующую исправления.

Дополнительную информацию о гидратации можно найти в [данном руководстве](guide/hydration).

Следующий пример вызовет эту ошибку.

```angular-ts
@Component({
  selector: 'app-example',
  template: '<div><span>world</span></div>',
})
export class Example {
  hostElement = inject(ElementRef).nativeElement;

  ngOnInit() {
    // Создаём новый элемент <p> с текстом `Hello` внутри
    const newNode = document.createElement('p');
    newNode.innerHTML = 'Hello';

    // Вставляем <p> перед первым элементом. Поскольку Angular не знает
    // об элементе <p>, он будет искать элемент <div> на первой позиции.
    // В результате будет выброшена ошибка несоответствия гидратации.
    // Вместо этого обновите шаблон компонента для создания элемента <p>.
    this.hostElement.insertBefore(newNode, this.hostElement.firstChild);
  }
}
```

## Отладка ошибки {#debugging-the-error}

Сообщение об ошибке в консоли разработчика должно содержать информацию о конкретном фрагменте DOM-структуры приложения, вызывающем проблему. Проверьте этот фрагмент на наличие ошибок, связанных с гидратацией, например прямого манипулирования DOM с помощью нативных API.

Убедитесь, что шаблон имеет корректную HTML-структуру. Дополнительную информацию см. в [руководстве по гидратации](guide/hydration#valid-html-structure).

В качестве возможного обходного пути можно добавить атрибут `ngSkipHydration` к хост-узлу компонента.

Если приложение работает в среде разработки, но в production-сборках возникают ошибки гидратации, убедитесь, что HTML, отправляемый браузеру, содержит комментарии, созданные Angular в процессе серверного рендеринга. Эти комментарии используются Angular в качестве якорей для контейнеров представлений (с гидратацией и без неё), и процесс гидратации предполагает их наличие на исходных позициях. Если у вас есть собственная логика для удаления комментариев из HTML, сгенерированного при серверном рендеринге, или если CDN настроен на их удаление перед отдачей контента, — отключите удаление комментариев и проверьте, исчезнут ли ошибки гидратации.
