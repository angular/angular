# Несоответствие узлов при гидратации

Эта ошибка означает, что в процессе гидратации Angular ожидал DOM-структуру, отрендеренную и аннотированную во время
рендеринга на стороне сервера. Однако на клиенте DOM-дерево отличалось от DOM-дерева, сгенерированного сервером.

Эта ошибка обычно возникает из-за прямых манипуляций с DOM с помощью нативных API браузера, которые изменяют структуру
DOM за пределами того, что сгенерировал Angular. Она также возникает, если вы используете `innerHTML` или `outerHTML`
для установки HTML-контента, что аналогичным образом изменяет структуру DOM. Вы можете решить эту проблему,
отрефакторив компонент для использования нативных Angular API вместо нативных браузерных. Если это невозможно, вы
можете добавить атрибут `ngSkipHydration` к хост-узлу вашего компонента, что отключит гидратацию для компонента и его
дочерних элементов. `ngSkipHydration` следует использовать только как крайнюю меру, и его применение следует
рассматривать как баг, требующий исправления.

Подробнее о гидратации можно узнать в [этом руководстве](guide/hydration).

Следующий пример вызовет ошибку.

```angular-ts
@Component({
  selector: 'app-example',
  template: '<div><span>world</span></div>',
})
export class Example {
  hostElement = inject(ElementRef).nativeElement;

  ngOnInit() {
    // Создаем новый элемент <p> с текстом 'Hello' внутри
    const newNode = document.createElement('p');
    newNode.innerHTML = 'Hello';

    // Вставляем <p> перед первым элементом. Поскольку Angular не имеет информации
    // об элементе <p>, он будет искать элемент <div> на первой позиции.
    // В результате будет выброшена ошибка несоответствия гидратации.
    // Вместо этого обновите шаблон компонента для создания элемента <p>.
    this.hostElement.insertBefore(newNode, this.hostElement.firstChild);
  }
}
```

## Отладка ошибки {#debugging-the-error}

Сообщение об ошибке в консоли разработчика должно содержать информацию о конкретной части DOM-структуры приложения,
вызывающей проблему. Проверьте эту часть приложения на наличие ошибок, связанных с гидратацией, например прямых
манипуляций с DOM с помощью нативных API.

Убедитесь, что ваш шаблон имеет корректную HTML-структуру. Подробнее см. в
[руководстве по гидратации](guide/hydration#valid-html-structure).

Вы можете добавить атрибут `ngSkipHydration` к хост-узлу вашего компонента в качестве возможного обходного решения.

Если ваше приложение работает в среде разработки, но вы видите ошибки гидратации в production-сборках, убедитесь, что
сгенерированный HTML, доставляемый в браузер, содержит узлы комментариев, создаваемые Angular во время рендеринга на
сервере. Эти узлы комментариев используются Angular runtime в качестве якорей для контейнеров представлений (с
гидратацией и без неё), и процесс гидратации ожидает их наличия на исходных позициях. Если у вас есть кастомная логика
для удаления узлов комментариев из HTML, сгенерированного при рендеринге на стороне сервера, или если вы настроили CDN
для их удаления перед отдачей контента — отключите удаление узлов комментариев и проверьте, исчезли ли ошибки
гидратации.
