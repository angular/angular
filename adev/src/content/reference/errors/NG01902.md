# Orphan field в signal forms

Эта ошибка означает, что экземпляр `Field` больше не связан с текущей структурой модели.
Angular signal forms выбрасывают `NG01902`, когда путь поля, который ранее существовал, больше не может быть разрешён как то же свойство в родительском объекте.

Частый триггер — непреднамеренная установка значения поля в `undefined`.
В signal forms `undefined` означает «поле не существует», поэтому существующее поле может стать orphaned.

## Почему это происходит {#why-this-happens}

Signal forms синхронизируют экземпляры полей с формой вашей модели.
Если код всё ещё ссылается на старый экземпляр поля после изменения формы модели, Angular обнаруживает несоответствие и выбрасывает эту ошибку.

Это может произойти, когда:

- свойство удаляется из объекта модели
- свойство становится `undefined` вместо того, чтобы оставаться присутствующим
- код удерживает устаревшую ссылку на поле между структурными обновлениями

## Как исправить {#how-to-fix-it}

- Избегайте записи `undefined` в поля модели. Используйте `null` или другое явное значение, когда поле должно продолжать существовать.
- Сохраняйте форму модели стабильной, пока поле используется.
- После структурных обновлений заново читайте поля из текущего дерева формы вместо повторного использования устаревших ссылок.

Руководство по проектированию модели см. в [Избегайте `undefined`](/guide/forms/signals/model-design#avoid-undefined).
