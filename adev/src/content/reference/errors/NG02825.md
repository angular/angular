# Тело ответа Fetch превышает настроенный лимит

Angular выдаёт эту ошибку, когда `HttpClient` использует fetch backend во время SSR и байты, буферизуемые для тела ответа, превышают настроенный лимит `maxResponseBodySize`. По умолчанию Angular ограничивает каждое буферизуемое тело ответа 1 МБ, чтобы предотвратить буферизацию сервером неожиданно больших ответов во время рендеринга.

## Исправление ошибки {#fixing-the-error}

Установите `maxResponseBodySize` в опциях `provideServerRendering`, чтобы увеличить лимит:

```ts
import {provideServerRendering, withRoutes} from '@angular/ssr';
import {serverRoutes} from './app.routes.server';

const serverConfig: ApplicationConfig = {
  providers: [
    provideServerRendering(
      {
        maxResponseBodySize: 5 * 1024 * 1024, // 5MB
      },
      withRoutes(serverRoutes),
    ),
  ],
};
```

`maxResponseBodySize` задаёт максимальное число байт тела ответа, которые Angular буферизует для каждого серверного запроса `HttpClient`, использующего fetch backend. Значение задаётся в байтах и применяется глобально.

IMPORTANT: Держите этот лимит настолько малым, насколько позволяет приложение. Его увеличение позволяет серверным запросам буферизовать более крупные тела ответов, что может увеличить использование памяти и риск denial-of-service. Предпочитайте выносить большие загрузки за пределы server rendering.

## Отладка ошибки {#debugging-the-error}

Сообщение об ошибке включает настроенный лимит `maxResponseBodySize` в байтах. Чтобы определить, какой HTTP-запрос возвращает большой ответ, проверьте сетевые логи или консоль SSR-сервера на URL неудачного запроса.

Если большой ответ нужен только на клиенте и не требуется для SSR, можно отключить transfer cache для этого конкретного запроса:

```ts
httpClient.get('/api/large-data', {transferCache: false});
```
