# Bootstrapped NgModule doesn't specify which component to initialize {#bootstrapped-ngmodule-doesnt-specify-which-component-to-initialize}

Эта ошибка означает, что NgModule, используемый для начальной загрузки приложения, не содержит необходимой информации для продолжения процесса начальной загрузки.

Ошибка возникает, когда свойство `bootstrap` NgModule отсутствует (или является пустым массивом) в аннотации `@NgModule`, и при этом на классе NgModule не определён хук жизненного цикла `ngDoBootstrap`.

Подробнее о процессе начальной загрузки можно узнать в [этом руководстве](guide/ngmodules/overview).

Следующие примеры вызовут ошибку.

```typescript
@NgModule({
  declarations: [App],
  imports: [BrowserModule, AppRoutingModule],
  providers: [],
})
export class AppModule {}

// `AppModule` используется для начальной загрузки, но поле `@NgModule.bootstrap` отсутствует.
platformBrowser().bootstrapModule(AppModule);
```

```typescript
@NgModule({
  declarations: [App],
  imports: [BrowserModule, AppRoutingModule],
  providers: [],
  bootstrap: [],
})
export class AppModule {}

// `AppModule` используется для начальной загрузки, но поле `@NgModule.bootstrap` содержит пустой массив.
platformBrowser().bootstrapModule(AppModule);
```

## Отладка ошибки {#debugging-the-error}

Убедитесь, что NgModule, используемый для начальной загрузки, настроен корректно:

- либо свойство `bootstrap` существует (и содержит непустой массив) в аннотации `@NgModule`
- либо в классе NgModule существует метод `ngDoBootstrap`
