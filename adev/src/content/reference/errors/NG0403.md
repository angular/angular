# Загружаемый NgModule не указывает, какой компонент инициализировать

Эта ошибка означает, что NgModule, используемый для загрузки (bootstrap) приложения, не содержит ключевой информации,
необходимой Angular для продолжения процесса загрузки.

Ошибка возникает, когда свойство `bootstrap` в аннотации `@NgModule` отсутствует (или является пустым массивом) и на
классе NgModule не определен хук жизненного цикла `ngDoBootstrap`.

Подробнее о процессе загрузки можно узнать в [этом руководстве](guide/ngmodules/overview).

Следующие примеры вызовут ошибку.

```typescript
@NgModule({
  declarations: [App],
  imports: [BrowserModule, AppRoutingModule],
  providers: [],
})
export class AppModule {}

// `AppModule` используется для загрузки, но поле `@NgModule.bootstrap` отсутствует.
platformBrowser().bootstrapModule(AppModule);
```

```typescript
@NgModule({
  declarations: [App],
  imports: [BrowserModule, AppRoutingModule],
  providers: [],
  bootstrap: [],
})
export class AppModule {}

// `AppModule` используется для загрузки, но поле `@NgModule.bootstrap` содержит пустой массив.
platformBrowser().bootstrapModule(AppModule);
```

## Отладка ошибки {#debugging-the-error}

Убедитесь, что NgModule, используемый для загрузки, настроен корректно:

- либо свойство `bootstrap` существует (и содержит непустой массив) в аннотации `@NgModule`
- либо на классе NgModule существует метод `ngDoBootstrap`
