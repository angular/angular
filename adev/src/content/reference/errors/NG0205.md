# Injector has already been destroyed {#injector-has-already-been-destroyed}

Эта ошибка возникает при попытке получить сервис из инжектора, который уже был уничтожен. Обычно это происходит, когда код пытается получить доступ к зависимостям после уничтожения компонента, директивы или модуля.

## Типичные сценарии {#common-scenarios}

### Обращение к сервисам в обратных вызовах после уничтожения {#accessing-services-in-callbacks-after-destruction}

При уничтожении компонента его инжектор также уничтожается. Если асинхронный обратный вызов впоследствии пытается получить доступ к сервисам, возникает эта ошибка:

```ts
@Component({
  /*...*/
})
export class UserProfile implements OnInit {
  private userClient = inject(UserClient);

  ngOnInit() {
    setTimeout(() => {
      // ОШИБКА: если компонент был уничтожен до срабатывания таймера,
      // инжектор уже недоступен
      this.userClient.fetchData();
    }, 5000);
  }
}
```

### Обращение к сервисам после отписки {#accessing-services-after-unsubscribing}

Аналогичные проблемы возникают с Observable, если очистка происходит в неправильном порядке:

```ts
@Component({
  /*...*/
})
export class DataView implements OnDestroy {
  private dataStore = inject(DataStore);

  ngOnDestroy() {
    // Проблема: попытка использовать инжектор во время уничтожения
    // после того, как другая очистка уже могла произойти
    this.dataStore.cleanup();
  }
}
```

## Отладка ошибки {#debugging-the-error}

Для исправления ошибки:

1. **Проверьте асинхронные операции** — убедитесь, что обратные вызовы, промисы и подписки отменяются при уничтожении компонента. Используйте `takeUntilDestroyed()` или `DestroyRef` для очистки.

2. **Захватывайте зависимости заранее** — сохраняйте ссылки на сервисы в полях класса, а не обращайтесь к инжектору в обратных вызовах.

3. **Защищайтесь от уничтоженного состояния** — для операций, которые могут пережить компонент, проверяйте, активен ли компонент, прежде чем обращаться к сервисам.

```ts
@Component({
  /*...*/
})
export class UserProfile implements OnInit {
  private destroyRef = inject(DestroyRef);
  private userClient = inject(UserClient);

  ngOnInit() {
    // Используйте takeUntilDestroyed для автоматической отмены при уничтожении
    interval(5000)
      .pipe(takeUntilDestroyed(this.destroyRef))
      .subscribe(() => {
        this.userClient.fetchData();
      });
  }
}
```

Стек вызовов указывает место, где был обращён уничтоженный инжектор. Проследите назад, чтобы найти асинхронную операцию, пережившую компонент.
