# Инжектор уже уничтожен

Эта ошибка возникает при попытке получить сервис из инжектора, который уже был уничтожен. Обычно это происходит, когда
код пытается получить доступ к зависимостям после уничтожения компонента, директивы или модуля.

## Распространенные сценарии {#common-scenarios}

### Обращение к сервисам в колбэках после уничтожения {#accessing-services-in-callbacks-after-destruction}

При уничтожении компонента его инжектор также уничтожается. Если асинхронный колбэк впоследствии попытается получить
доступ к сервисам, возникнет эта ошибка:

```ts
@Component({
  /*...*/
})
export class UserProfile implements OnInit {
  private userClient = inject(UserClient);

  ngOnInit() {
    setTimeout(() => {
      // ОШИБКА: если компонент был уничтожен до срабатывания таймаута,
      // инжектор больше недоступен
      this.userClient.fetchData();
    }, 5000);
  }
}
```

### Обращение к сервисам после отписки {#accessing-services-after-unsubscribing}

Аналогичные проблемы возникают с observable, если очистка происходит в неправильном порядке:

```ts
@Component({
  /*...*/
})
export class DataView implements OnDestroy {
  private dataStore = inject(DataStore);

  ngOnDestroy() {
    // Проблемный код: попытка использовать инжектор во время уничтожения,
    // после того как другая очистка уже могла произойти
    this.dataStore.cleanup();
  }
}
```

## Отладка ошибки {#debugging-the-error}

Для исправления этой ошибки:

1. **Проверьте асинхронные операции** — убедитесь, что колбэки, промисы и подписки отменяются при уничтожении
   компонента. Используйте `takeUntilDestroyed()` или `DestroyRef` для очистки.

2. **Получайте зависимости заранее** — сохраняйте ссылки на сервисы в полях класса, а не обращайтесь к инжектору в
   колбэках.

3. **Защищайтесь от уничтоженного состояния** — для операций, которые могут пережить компонент, проверяйте, активен ли
   компонент, перед обращением к сервисам.

```ts
@Component({
  /*...*/
})
export class UserProfile implements OnInit {
  private destroyRef = inject(DestroyRef);
  private userClient = inject(UserClient);

  ngOnInit() {
    // Используйте takeUntilDestroyed для автоматической отмены при уничтожении
    interval(5000)
      .pipe(takeUntilDestroyed(this.destroyRef))
      .subscribe(() => {
        this.userClient.fetchData();
      });
  }
}
```

Стек вызовов указывает место, где был использован уничтоженный инжектор. Пройдите вверх по стеку, чтобы найти
асинхронную операцию, которая пережила свой компонент.
