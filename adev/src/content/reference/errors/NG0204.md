# Invalid Injection Token {#invalid-injection-token}

Эта ошибка возникает, когда Angular не может разрешить зависимость для класса в процессе внедрения зависимостей. Чаще всего она затрагивает классы, использующие инъекцию через конструктор, где Angular полагается на метаданные TypeScript для определения типов параметров.

Наиболее распространённые причины:

1. У класса сервиса отсутствует декоратор `@Injectable()`
2. `InjectionToken` не имеет корректного определения провайдера
3. Параметр конструктора не может быть разрешён

ПРИМЕЧАНИЕ: Функция `inject()` принимает явный токен, поэтому сценарий «неразрешимого параметра» к ней напрямую не относится. Однако если у самого внедряемого класса отсутствует `@Injectable()` и он имеет собственные зависимости в конструкторе, ошибка всё равно может возникнуть.

## Типичные сценарии {#common-scenarios}

### Отсутствует декоратор `@Injectable()` {#missing-injectable-decorator}

Если у класса есть зависимости в конструкторе, но отсутствует декоратор `@Injectable()`, Angular не сможет разрешить его параметры:

```ts {header: 'Отсутствует декоратор @Injectable()'}
export class UserClient {
  constructor(private http: HttpClient) {} // Angular не может разрешить это
}
```

Добавьте декоратор `@Injectable()` для исправления:

```ts
@Injectable({providedIn: 'root'})
export class UserClient {
  constructor(private http: HttpClient) {}
}
```

### Неразрешимые параметры конструктора {#unresolvable-constructor-parameters}

Ошибка также появляется, когда Angular не может определить тип параметра конструктора:

```ts
@Injectable({providedIn: 'root'})
export class DataStore {
  // Angular не может разрешить 'config' без провайдера
  constructor(private config: AppConfig) {}
}
```

Убедитесь, что все параметры конструктора либо имеют настроенные провайдеры, либо используют `@Optional()` для необязательных зависимостей.

## Отладка ошибки {#debugging-the-error}

Сообщение об ошибке содержит подробности о том, какой токен не удалось разрешить:

- `Can't resolve all parameters for X: (?, ?, ?)` — знаки `?` указывают на неразрешимые параметры. Убедитесь, что у класса есть `@Injectable()` и у всех зависимостей есть провайдеры.
- `Token X is missing a ɵprov definition` — `InjectionToken` использован без настроенного провайдера. Зарегистрируйте токен со значением через `{provide: TOKEN, useValue: ...}` или добавьте фабрику по умолчанию в определение токена.

Проследите по стеку вызовов ошибки назад, чтобы найти место проблемной инъекции, затем убедитесь, что:

1. У класса есть декоратор `@Injectable()`
2. Для всех параметров конструктора зарегистрированы провайдеры
3. Любой `InjectionToken` имеет настроенный провайдер или значение по умолчанию
