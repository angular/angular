# Недопустимый токен внедрения

Эта ошибка возникает, когда Angular не может разрешить зависимость для класса во время внедрения зависимостей. Чаще
всего она затрагивает классы, использующие внедрение через конструктор, где Angular полагается на метаданные TypeScript
для определения типов параметров.

Наиболее распространенные причины:

1. У класса сервиса отсутствует декоратор `@Injectable()`
2. `InjectionToken` не имеет определения провайдера
3. Параметр конструктора не может быть разрешен

ПРИМЕЧАНИЕ: Функция `inject()` принимает явный токен, поэтому сценарий "неразрешимого параметра" к ней напрямую не
применяется. Однако если у внедряемого класса отсутствует `@Injectable()` и у него есть собственные зависимости в
конструкторе, ошибка всё равно может возникнуть.

## Распространенные сценарии {#common-scenarios}

### Отсутствует декоратор `@Injectable()` {#missing-injectable-decorator}

Если у класса есть зависимости в конструкторе, но отсутствует декоратор `@Injectable()`, Angular не сможет разрешить
его параметры:

```ts {header: 'Отсутствует декоратор @Injectable()'}
export class UserClient {
  constructor(private http: HttpClient) {} // Angular не может разрешить это
}
```

Добавьте декоратор `@Injectable()` для исправления:

```ts
@Injectable({providedIn: 'root'})
export class UserClient {
  constructor(private http: HttpClient) {}
}
```

### Неразрешимые параметры конструктора {#unresolvable-constructor-parameters}

Эта ошибка также возникает, когда Angular не может определить тип параметра конструктора:

```ts
@Injectable({providedIn: 'root'})
export class DataStore {
  // Angular не может разрешить 'config' без провайдера
  constructor(private config: AppConfig) {}
}
```

Убедитесь, что все параметры конструктора либо имеют настроенные провайдеры, либо используют `@Optional()` для
необязательных зависимостей.

## Отладка ошибки {#debugging-the-error}

Сообщение об ошибке содержит подробности о том, какой токен не удалось разрешить:

- `Can't resolve all parameters for X: (?, ?, ?)` — знаки `?` указывают на неразрешимые параметры. Убедитесь, что у
  класса есть `@Injectable()` и для всех зависимостей настроены провайдеры.
- `Token X is missing a ɵprov definition` — `InjectionToken` использован без настройки провайдера. Зарегистрируйте
  токен со значением через `{provide: TOKEN, useValue: ...}` или добавьте фабрику по умолчанию в определение токена.

Пройдите вверх по стеку вызовов ошибки, чтобы найти место, где происходит проблемное внедрение, и убедитесь, что:

1. У класса есть декоратор `@Injectable()`
2. Все параметры конструктора имеют зарегистрированные провайдеры
3. Для любого `InjectionToken` настроен провайдер или значение по умолчанию
