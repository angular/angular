# Некорректный Injection Token

Эта ошибка возникает, когда Angular не может разрешить зависимость для класса во время внедрения зависимостей. Чаще всего это затрагивает классы, использующие constructor injection, где Angular опирается на метаданные TypeScript для определения типов параметров.

Самые частые причины:

1. У класса сервиса отсутствует декоратор `@Service()` или `@Injectable`
2. У `InjectionToken` нет корректного определения provider
3. Параметр конструктора не может быть разрешён

NOTE: Функция `inject()` принимает явный токен, поэтому сценарий «неразрешимый параметр» к ней напрямую не применяется. Однако если у внедряемого класса отсутствует `@Service()` и у него есть собственные зависимости конструктора, ошибка всё равно может возникнуть.

## Типичные сценарии {#common-scenarios}

### Отсутствует декоратор `@Service()` {#missing-service-decorator}

Когда у класса есть зависимости конструктора, но нет декоратора `@Service()`, Angular не может разрешить его параметры:

```ts {header: 'Missing @Service() decorator'}
export class UserClient {
  constructor(private http: HttpClient) {} // Angular can't resolve this
}
```

Добавьте декоратор `@Service()`, чтобы исправить это:

```ts
@Service()
export class UserClient {
  constructor(private http: HttpClient) {}
}
```

### Неразрешимые параметры конструктора {#unresolvable-constructor-parameters}

Эта ошибка также появляется, когда Angular не может определить тип параметра конструктора:

```ts
@Service()
export class DataStore {
  // Angular can't resolve 'config' without a provider
  constructor(private config: AppConfig) {}
}
```

Убедитесь, что у всех параметров конструктора настроены providers или используется `@Optional()` для необязательных зависимостей.

## Отладка ошибки {#debugging-the-error}

Сообщение об ошибке содержит сведения о том, какой токен не удалось разрешить:

- `Can't resolve all parameters for X: (?, ?, ?)` — знаки `?` указывают на неразрешимые параметры. Проверьте, что у класса есть `@Service` или `@Injectable()` и у всех зависимостей есть providers.
- `Token X is missing a ɵprov definition` — `InjectionToken` использован без настройки provider. Зарегистрируйте токен со значением через `{provide: TOKEN, useValue: ...}` или добавьте default factory в определение токена.

Идите назад по stack trace ошибки, чтобы определить, где происходит проблемное внедрение, затем проверьте, что:

1. У класса есть декоратор `@Service` или `@Injectable()`
2. У всех параметров конструктора зарегистрированы providers
3. У любого `InjectionToken` настроен provider или значение по умолчанию
