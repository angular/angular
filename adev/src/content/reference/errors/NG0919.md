# Обнаружена циклическая зависимость

Angular обнаружил циклическую зависимость между компонентами, директивами или pipes. Эта ошибка возникает, когда компонент A импортирует компонент B, а компонент B (напрямую или косвенно) импортирует компонент A, создавая цикл.

Эта циклическая ссылка мешает Angular правильно инициализировать компоненты, что приводит к ошибке вроде:

```text
NG0919: Cannot read @Component metadata. This can indicate a runtime circular dependency in your app that needs to be resolved.
```

В более старых версиях Angular вместо этого можно увидеть ошибку вроде:

```text
Cannot read properties of undefined (reading 'ɵcmp')
```

## Типичные причины {#common-causes}

### Взаимные импорты компонентов {#mutual-component-imports}

Самая частая причина — когда два компонента импортируют друг друга:

```angular-ts {header:"parent.ts"}
import {Component} from '@angular/core';
import {Child} from './child';

@Component({
  selector: 'app-parent',
  imports: [Child],
  template: '<app-child/>',
})
export class Parent {}
```

```angular-ts {header:"child.ts"}
import {Component} from '@angular/core';
import {Parent} from './parent';

@Component({
  selector: 'app-child',
  imports: [Parent],
  template: '<app-parent/>',
})
export class Child {}
```

### Косвенные циклические ссылки {#indirect-circular-references}

Циклические зависимости также могут возникать через промежуточные файлы:

```text
ComponentA -> ComponentB -> ComponentC -> ComponentA
```

## Разрешение ошибки {#resolving-the-error}

### Рефакторинг общей логики {#refactor-shared-logic}

Перенесите общую функциональность в отдельный файл, который не импортирует ни один из компонентов:

```ts {header:"shared.service.ts"}
import {Service} from '@angular/core';

@Service()
export class SharedService {
  // Shared logic here
}
```

### Используйте type-only импорты {#use-type-only-imports}

Если нужны только типы для TypeScript, используйте `import type`:

```ts
import type {Parent} from './parent';
```

Type-only импорты удаляются во время компиляции и не участвуют в runtime циклических зависимостях.

### Перестройте иерархию компонентов {#restructure-component-hierarchy}

Подумайте, не указывает ли циклическая зависимость на проблему дизайна. Часто самым чистым решением является извлечение общей функциональности в третий компонент или сервис.

### Отладка сложных циклических зависимостей {#debugging-complex-circular-dependencies}

В сложных приложениях с множеством модулей циклические зависимости бывает трудно выявить вручную. Рассмотрите использование инструментов вроде [madge](https://www.npmjs.com/package/madge) для визуализации и обнаружения циклических импортов:

```bash
# Install madge
npm install -g madge

# Check for circular dependencies
madge --circular --extensions ts ./src

# Generate a visual graph
madge --circular --extensions ts --image graph.svg ./src
```

Эти инструменты помогают выявить цепочки циклических зависимостей по всему проекту и генерировать визуальные графы зависимостей.
