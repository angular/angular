# Circular Dependency Detected {#circular-dependency-detected}

Angular обнаружил циклическую зависимость между компонентами, директивами или Pipe. Эта ошибка возникает, когда компонент A импортирует компонент B, а компонент B (напрямую или косвенно) импортирует компонент A, создавая цикл.

Эта циклическая ссылка мешает Angular правильно инициализировать компоненты, что приводит к ошибке вида:

```text
NG0919: Cannot read @Component metadata. This can indicate a runtime circular dependency in your app that needs to be resolved.
```

В старых версиях Angular вместо этого можно было увидеть ошибку:

```text
Cannot read properties of undefined (reading 'ɵcmp')
```

## Типичные причины {#common-causes}

### Взаимный импорт компонентов {#mutual-component-imports}

Наиболее распространённая причина — когда два компонента импортируют друг друга:

```angular-ts {header:"parent.ts"}
import {Component} from '@angular/core';
import {Child} from './child';

@Component({
  selector: 'app-parent',
  imports: [Child],
  template: '<app-child/>',
})
export class Parent {}
```

```angular-ts {header:"child.ts"}
import {Component} from '@angular/core';
import {Parent} from './parent';

@Component({
  selector: 'app-child',
  imports: [Parent],
  template: '<app-parent/>',
})
export class Child {}
```

### Косвенные циклические ссылки {#indirect-circular-references}

Циклические зависимости могут также возникать через промежуточные файлы:

```text
ComponentA -> ComponentB -> ComponentC -> ComponentA
```

## Устранение ошибки {#resolving-the-error}

### Вынесите общую логику {#refactor-shared-logic}

Переместите общую функциональность в отдельный файл, который не импортирует ни один из компонентов:

```angular-ts {header:"shared.service.ts"}
import {Injectable} from '@angular/core';

@Injectable({providedIn: 'root'})
export class SharedService {
  // Общая логика здесь
}
```

### Используйте импорты только типов {#use-type-only-imports}

Если вам нужны только типы TypeScript, используйте `import type`:

```ts
import type {Parent} from './parent';
```

Импорты только типов стираются во время компиляции и не способствуют циклическим зависимостям во время выполнения.

### Реструктурируйте иерархию компонентов {#restructure-component-hierarchy}

Подумайте, не указывает ли циклическая зависимость на проблему проектирования. Часто наиболее чистым решением является вынесение общей функциональности в третий компонент или сервис.

### Отладка сложных циклических зависимостей {#debugging-complex-circular-dependencies}

Для сложных приложений с множеством модулей циклические зависимости бывает трудно определить вручную. Рассмотрите использование инструментов вроде [madge](https://www.npmjs.com/package/madge) для визуализации и обнаружения циклических импортов:

```bash
# Установить madge
npm install -g madge

# Проверить на наличие циклических зависимостей
madge --circular --extensions ts ./src

# Создать визуальный граф
madge --circular --extensions ts --image graph.svg ./src
```

Эти инструменты помогут выявить цепочки циклических зависимостей во всём проекте и создать визуальные графы зависимостей.
