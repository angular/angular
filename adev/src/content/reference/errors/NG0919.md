# Обнаружена циклическая зависимость

Angular обнаружил циклическую зависимость между компонентами, директивами или pipe. Эта ошибка возникает, когда
компонент A импортирует компонент B, а компонент B (напрямую или косвенно) импортирует компонент A, создавая цикл.

Эта циклическая ссылка препятствует корректной инициализации компонентов, что приводит к ошибке вида:

```text
NG0919: Cannot read @Component metadata. This can indicate a runtime circular dependency in your app that needs to be resolved.
```

В старых версиях Angular вместо этого можно было видеть ошибку вида:

```text
Cannot read properties of undefined (reading 'ɵcmp')
```

## Распространенные причины {#common-causes}

### Взаимный импорт компонентов {#mutual-component-imports}

Наиболее распространенная причина — два компонента, импортирующих друг друга:

```angular-ts {header:"parent.ts"}
import {Component} from '@angular/core';
import {Child} from './child';

@Component({
  selector: 'app-parent',
  imports: [Child],
  template: '<app-child/>',
})
export class Parent {}
```

```angular-ts {header:"child.ts"}
import {Component} from '@angular/core';
import {Parent} from './parent';

@Component({
  selector: 'app-child',
  imports: [Parent],
  template: '<app-parent/>',
})
export class Child {}
```

### Косвенные циклические ссылки {#indirect-circular-references}

Циклические зависимости также могут возникать через промежуточные файлы:

```text
ComponentA -> ComponentB -> ComponentC -> ComponentA
```

## Устранение ошибки {#resolving-the-error}

### Рефакторинг общей логики {#refactor-shared-logic}

Перенесите общую функциональность в отдельный файл, который не импортирует ни один из компонентов:

```angular-ts {header:"shared.service.ts"}
import {Injectable} from '@angular/core';

@Injectable({providedIn: 'root'})
export class SharedService {
  // Общая логика здесь
}
```

### Используйте импорт только типов {#use-type-only-imports}

Если вам нужны только типы для TypeScript, используйте `import type`:

```ts
import type {Parent} from './parent';
```

Импорты только типов удаляются во время компиляции и не способствуют возникновению циклических зависимостей во время
выполнения.

### Реструктуризация иерархии компонентов {#restructure-component-hierarchy}

Рассмотрите, не указывает ли циклическая зависимость на проблему в дизайне. Зачастую наиболее чистым решением является
извлечение общей функциональности в третий компонент или сервис.

### Отладка сложных циклических зависимостей {#debugging-complex-circular-dependencies}

В сложных приложениях с множеством модулей циклические зависимости может быть трудно выявить вручную. Рассмотрите
возможность использования инструментов вроде [madge](https://www.npmjs.com/package/madge) для визуализации и
обнаружения циклических импортов:

```bash
# Установка madge
npm install -g madge

# Проверка на наличие циклических зависимостей
madge --circular --extensions ts ./src

# Генерация визуального графа
madge --circular --extensions ts --image graph.svg ./src
```

Эти инструменты помогут найти цепочки циклических зависимостей во всём проекте и сгенерировать визуальные графы
зависимостей.
