# `inject()` должен вызываться из контекста внедрения

Эта ошибка возникает при попытке использовать функцию [`inject`](api/core/inject) за пределами допустимого
[контекста внедрения](guide/di/dependency-injection-context). Контекст внедрения доступен во время создания и
инициализации класса, а также в функциях, используемых с `runInInjectionContext`.

На практике вызовы `inject()` разрешены в конструкторе, параметрах конструктора и инициализаторах полей:

```typescript
@Injectable({providedIn: 'root'})
export class Car {
  radio: Radio | undefined;

  // OK: инициализатор поля
  spareTyre = inject(Tyre);

  constructor() {
    // OK: тело конструктора
    this.radio = inject(Radio);
  }
}
```

Также допустимо вызывать [`inject`](api/core/inject) из фабрики провайдера:

```typescript
providers: [
  {
    provide: Car,
    useFactory: () => {
      // OK: фабрика класса
      const engine = inject(Engine);
      return new Car(engine);
    },
  },
];
```

Вызовы функции [`inject`](api/core/inject) за пределами создания класса или `runInInjectionContext` приведут к ошибке.
В частности, вызовы `inject()` недопустимы после создания экземпляра класса — в методах (включая хуки жизненного цикла):

```typescript
@Component({ ... })
export class Car {
  ngOnInit() {
    // ОШИБКА: слишком поздно, экземпляр компонента уже создан
    const engine = inject(Engine);
    engine.start();
  }
}
```

## Отладка ошибки {#debugging-the-error}

Пройдите вверх по стеку вызовов ошибки, чтобы найти место, где находится недопустимый вызов `inject()`.

Чтобы исправить ошибку, переместите вызов [`inject`](api/core/inject) в допустимое место (обычно в конструктор класса
или инициализатор поля).

**ПРИМЕЧАНИЕ:** При работе в тестовом контексте `TestBed.runInInjectionContext` позволит `inject()` выполниться успешно.

```typescript
TestBed.runInInjectionContext(() => {
  // ...
});
```
