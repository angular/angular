# `inject()` must be called from an injection context {#inject-must-be-called-from-an-injection-context}

Эта ошибка возникает, когда вы пытаетесь использовать функцию [`inject`](api/core/inject) за пределами допустимого [контекста внедрения зависимостей](guide/di/dependency-injection-context). Контекст внедрения доступен во время создания и инициализации класса, а также в функциях, используемых с `runInInjectionContext`.

На практике вызовы `inject()` допустимы в конструкторе, параметрах конструктора и инициализаторах полей:

```typescript
@Injectable({providedIn: 'root'})
export class Car {
  radio: Radio | undefined;

  // OK: инициализатор поля
  spareTyre = inject(Tyre);

  constructor() {
    // OK: тело конструктора
    this.radio = inject(Radio);
  }
}
```

Вызов [`inject`](api/core/inject) из фабрики провайдера также допустим:

```typescript
providers: [
  {
    provide: Car,
    useFactory: () => {
      // OK: фабрика класса
      const engine = inject(Engine);
      return new Car(engine);
    },
  },
];
```

Вызовы функции [`inject`](api/core/inject) вне контекста создания класса или `runInInjectionContext` приведут к ошибке. В частности, вызовы `inject()` недопустимы после создания экземпляра класса, в методах (включая хуки жизненного цикла):

```typescript
@Component({ ... })
export class Car {
  ngOnInit() {
    // ОШИБКА: слишком поздно, экземпляр компонента уже создан
    const engine = inject(Engine);
    engine.start();
  }
}
```

## Отладка ошибки {#debugging-the-error}

Проследите по стеку вызовов назад, чтобы найти место, где расположен недопустимый вызов `inject()`.

Для исправления ошибки переместите вызов [`inject`](api/core/inject) в допустимое место (обычно в конструктор класса или инициализатор поля).

**ПРИМЕЧАНИЕ:** При работе в тестовом контексте `TestBed.runInInjectionContext` позволит `inject()` работать корректно.

```typescript
TestBed.runInInjectionContext(() => {
  // ...
});
```
