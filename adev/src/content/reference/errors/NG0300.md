# Selector Collision {#selector-collision}

<docs-video src="https://www.youtube.com/embed/z_3Z5mOm59I"/>

Два или более [компонента](guide/components) используют один и тот же селектор элемента. Поскольку с элементом может быть связан только один компонент, селекторы должны быть уникальными строками во избежание неоднозначности для Angular.

Эта ошибка возникает во время выполнения, когда вы применяете два селектора к одному узлу, каждый из которых соответствует разному компоненту, или когда вы применяете один селектор к узлу, и он соответствует более чем одному компоненту.

```angular-ts
import {Component} from '@angular/core';

@Component({
  selector: '[stroked-button]',
  templateUrl: './stroked-button.html',
})
export class StrokedBtn {}

@Component({
  selector: '[raised-button]',
  templateUrl: './raised-button.html',
})
export class RaisedBtn {}

@Component({
  selector: 'app-root',
  template: `
    <!-- У этого узла 2 селектора: stroked-button и raised-button, каждый из которых соответствует разному компоненту: StrokedBtnComponent и RaisedBtnComponent, поэтому будет поднята NG0300  -->

    <button stroked-button raised-button></button>
  `,
})
export class App {}
```

## Отладка ошибки {#debugging-the-error}

Используйте имя элемента из сообщения об ошибке для поиска мест, где вы используете одно и то же объявление селектора в кодовой базе:

```ts

@Component({
  selector: 'YOUR_STRING',
…
})

```

Убедитесь, что каждый компонент имеет уникальный CSS-селектор. Это гарантирует, что Angular отрендерит ожидаемый компонент.

Если вам сложно найти несколько компонентов с данным именем тега селектора, проверьте компоненты из импортированных библиотек, например Angular Material. Убедитесь, что вы следуете [рекомендациям](style-guide#choosing-component-selectors) по выбору селекторов, чтобы предотвратить коллизии.
