# Коллизия селекторов

<docs-video src="https://www.youtube.com/embed/z_3Z5mOm59I"/>

Два или более [компонента](guide/components) используют одинаковый селектор элемента. Поскольку с элементом может быть
связан только один компонент, селекторы должны быть уникальными строками во избежание неоднозначности для Angular.

Эта ошибка возникает во время выполнения, когда вы применяете два селектора к одному узлу, каждый из которых совпадает
с разным компонентом, или когда вы применяете один селектор к узлу и он совпадает более чем с одним компонентом.

```angular-ts
import {Component} from '@angular/core';

@Component({
  selector: '[stroked-button]',
  templateUrl: './stroked-button.html',
})
export class StrokedBtn {}

@Component({
  selector: '[raised-button]',
  templateUrl: './raised-button.html',
})
export class RaisedBtn {}

@Component({
  selector: 'app-root',
  template: `
    <!-- Этот узел имеет 2 селектора: stroked-button и raised-button,
         каждый из которых совпадает с другим компонентом: StrokedBtnComponent и RaisedBtnComponent,
         поэтому будет выброшена ошибка NG0300 -->

    <button stroked-button raised-button></button>
  `,
})
export class App {}
```

## Отладка ошибки {#debugging-the-error}

Используйте имя элемента из сообщения об ошибке для поиска мест, где в вашей кодовой базе используется одинаковое
объявление селектора:

```ts

@Component({
  selector: 'YOUR_STRING',
…
})

```

Убедитесь, что каждый компонент имеет уникальный CSS-селектор. Это гарантирует, что Angular отрендерит именно тот
компонент, который вы ожидаете.

Если вам сложно найти несколько компонентов с этим именем тега-селектора, проверьте компоненты из импортированных
библиотек компонентов, например Angular Material. Убедитесь, что вы следуете
[рекомендациям](style-guide#choosing-component-selectors) по выбору селекторов, чтобы избежать коллизий.
