# Предупреждения о производительности во время выполнения

### Изображения слишком большого размера {#oversized-images}

При загрузке изображений **исходный размер** загруженного файла сравнивается с фактическим размером изображения на
странице. Фактический размер вычисляется как **отрендеренный размер** изображения с применением CSS, умноженный на
[соотношение пикселей устройства](https://web.dev/codelab-density-descriptors/#pixel-density). Если загруженное
изображение значительно больше (более 1200 пикселей по любому из измерений), это предупреждение срабатывает. Загрузка
изображений слишком большого размера может замедлить загрузку страницы и негативно повлиять на
[Core Web Vitals](https://web.dev/vitals/).

### Лениво загружаемый LCP-элемент {#lazy-loaded-lcp-element}

Наибольший по содержимому элемент на странице во время загрузки считается "LCP-элементом", что связано с
[Largest Contentful Paint](https://web.dev/lcp/) — одним из Core Web Vitals. Ленивая загрузка LCP-элемента оказывает
значительное негативное влияние на производительность страницы. При такой стратегии браузер должен завершить расчеты
макета, чтобы определить, находится ли элемент в области видимости, прежде чем начать загрузку изображения. В
результате предупреждение срабатывает, когда Angular обнаруживает, что LCP-элемент имеет атрибут `loading="lazy"`.

### Отладка ошибки {#debugging-the-error}

Используйте URL изображения, указанный в предупреждении консоли, чтобы найти соответствующий элемент `<img>`.

### Способы исправления изображений слишком большого размера {#ways-to-fix-oversized-images}

- Используйте изображение меньшего размера в источнике
- Добавьте [`srcset`](https://web.dev/learn/design/responsive-images/#responsive-images-with-srcset), если для разных
  макетов нужны изображения разных размеров.
- Перейдите на использование встроенной директивы Angular для изображений
  ([`NgOptimizedImage`](/api/common/NgOptimizedImage)), которая [автоматически генерирует srcset](/guide/image-optimization#request-images-at-the-correct-size-with-automatic-srcset).

### Способы исправления лениво загружаемого LCP-элемента {#ways-to-fix-lazy-loaded-lcp-element}

- Измените атрибут `loading` на другое значение, например `"eager"`.
- Перейдите на использование встроенной директивы Angular для изображений
  ([`NgOptimizedImage`](/api/common/NgOptimizedImage)), которая позволяет легко
  [приоритизировать LCP-изображения](/guide/image-optimization#mark-images-as-priority).

### Отключение предупреждений о производительности изображений {#disabling-image-performance-warnings}

Оба предупреждения можно отключить по отдельности для всего сайта с помощью провайдера в корне приложения:

```typescript
providers: [
  {
    provide: IMAGE_CONFIG,
    useValue: {
      disableImageSizeWarning: true,
      disableImageLazyLoadWarning: true
    }
  },
],
```
