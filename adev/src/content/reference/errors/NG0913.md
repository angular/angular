# Runtime Performance Warnings {#runtime-performance-warnings}

### Слишком большие изображения {#oversized-images}

При загрузке изображений **внутренний размер** скачанного файла сравнивается с фактическим размером изображения на странице. Фактический размер рассчитывается исходя из **отрендеренного размера** изображения с применённым CSS, умноженного на [соотношение пикселей устройства](https://web.dev/codelab-density-descriptors/#pixel-density). Если скачанное изображение значительно больше (более чем на 1200px по любому из измерений), это предупреждение срабатывает. Загрузка слишком больших изображений может замедлить загрузку страницы и негативно сказаться на [Core Web Vitals](https://web.dev/vitals/).

### Ленивая загрузка LCP-элемента {#lazy-loaded-lcp-element}

Наибольший содержательный элемент на странице во время загрузки считается «LCP-элементом», что связано с [Largest Contentful Paint](https://web.dev/lcp/) — одним из Core Web Vitals. Ленивая загрузка LCP-элемента окажет значительное негативное влияние на производительность страницы. При такой стратегии браузер должен выполнить расчёты компоновки, чтобы определить, находится ли элемент в области видимости, прежде чем начать загрузку изображения. В результате предупреждение срабатывает, когда Angular обнаруживает, что LCP-элементу присвоен атрибут `loading="lazy"`.

### Отладка ошибки {#debugging-the-error}

Используйте URL изображения, указанный в предупреждении консоли, чтобы найти соответствующий элемент `<img>`.

### Способы исправления слишком больших изображений {#ways-to-fix-oversized-images}

- Используйте изображение меньшего размера
- Добавьте [`srcset`](https://web.dev/learn/design/responsive-images/#responsive-images-with-srcset), если для разных компоновок нужны изображения разных размеров.
- Перейдите на использование встроенной директивы Angular для изображений ([`NgOptimizedImage`](/api/common/NgOptimizedImage)), которая [автоматически генерирует srcset](/guide/image-optimization#request-images-at-the-correct-size-with-automatic-srcset).

### Способы исправления ленивой загрузки LCP-элемента {#ways-to-fix-lazy-loaded-lcp-element}

- Измените атрибут `loading` на другое значение, например `"eager"`.
- Перейдите на использование встроенной директивы Angular для изображений ([`NgOptimizedImage`](/api/common/NgOptimizedImage)), которая позволяет легко [устанавливать приоритет для LCP-изображений](/guide/image-optimization#mark-images-as-priority).

### Отключение предупреждений о производительности изображений {#disabling-image-performance-warnings}

Оба предупреждения можно отключить по отдельности для всего сайта с помощью провайдера в корне приложения:

```typescript
providers: [
  {
    provide: IMAGE_CONFIG,
    useValue: {
      disableImageSizeWarning: true,
      disableImageLazyLoadWarning: true
    }
  },
],
```
