# Обнаружен цикл импорта

Компонент, директива или pipe, на которые ссылается данный компонент, потребовали бы от компилятора добавить импорт,
который привел бы к циклу импортов. Например, рассмотрим сценарий, где `Parent` ссылается на `Child` в своем шаблоне:

```angular-ts {header:"parent.ts"}
import {Component} from '@angular/core';
import {Child} from './child';

@Component({
  selector: 'app-parent',
  imports: [Child],
  template: '<app-child/>',
})
export class Parent {}
```

```angular-ts {header:"child.ts"}
import {Component, inject} from '@angular/core';
import {Parent} from './parent';

@Component({
  selector: 'app-child',
  template: 'The child!',
})
export class Child {
  private parent = inject(Parent);
}
```

Из `child.ts` в `parent.ts` уже существует импорт, поскольку `Child` ссылается на `Parent` в своем конструкторе.
HELPFUL: Шаблон родительского компонента содержит `<child></child>`.
Сгенерированный код этого шаблона должен содержать ссылку на класс `Child`.
Чтобы создать эту ссылку, компилятору пришлось бы добавить импорт из `parent.ts` в `child.ts`, что создало бы цикл
импортов:

```text

parent.ts -> child.ts -> parent.ts

```

## Удаленная область видимости {#remote-scoping}

При использовании NgModules, чтобы избежать добавления импортов, создающих циклы, дополнительный код добавляется в
класс `NgModule`, в котором объявлен компонент, связывающий зависимости.

Это называется "удаленной областью видимости" (remote scoping).

## Библиотеки {#libraries}

К сожалению, код "удаленной области видимости" имеет побочные эффекты, что препятствует tree-shaking, и не может
использоваться в библиотеках. Поэтому при сборке библиотек с настройкой `"compilationMode": "partial"` любой
компонент, требующий циклического импорта, вызовет эту ошибку компилятора `NG3003`.

## Отладка ошибки {#debugging-the-error}

Цикл, который был бы создан, показан в сообщении об ошибке. Например:

```text {hideCopy}

The component Child is used in the template but importing it would create a cycle:
/parent.ts -> /child.ts -> /parent.ts

```

Используйте эту информацию, чтобы определить, как компонент, pipe или директива, на которые есть ссылка, создают
обратную зависимость на компилируемый компонент. Вот несколько идей для решения проблемы:

- Попробуйте перестроить зависимости, чтобы избежать цикла. Например, используя промежуточный интерфейс, хранящийся в
  независимом файле, который можно импортировать в оба зависимых файла без создания цикла импортов.
- Перенесите классы, ссылающиеся друг на друга, в один файл, чтобы избежать любых импортов между ними.
- Преобразуйте операторы импорта в импорты только типов \(используя синтаксис `import type`\), если импортируемые
  объявления используются только как типы — импорты только типов не способствуют образованию циклов.
