# Import Cycle Detected {#import-cycle-detected}

Компонент, директива или Pipe, на которые ссылается данный компонент, потребовали бы от компилятора добавления импорта, который привёл бы к циклу импортов.
Рассмотрим сценарий, где `Parent` ссылается на `Child` в своём шаблоне:

```angular-ts {header:"parent.ts"}
import {Component} from '@angular/core';
import {Child} from './child';

@Component({
  selector: 'app-parent',
  imports: [Child],
  template: '<app-child/>',
})
export class Parent {}
```

```angular-ts {header:"child.ts"}
import {Component, inject} from '@angular/core';
import {Parent} from './parent';

@Component({
  selector: 'app-child',
  template: 'The child!',
})
export class Child {
  private parent = inject(Parent);
}
```

Уже существует импорт из `child.ts` в `parent.ts`, поскольку `Child` ссылается на `Parent` в своём конструкторе.
ПОДСКАЗКА: Шаблон родительского компонента содержит `<child></child>`.
Сгенерированный код для этого шаблона должен поэтому содержать ссылку на класс `Child`.
Чтобы создать эту ссылку, компилятору пришлось бы добавить импорт из `parent.ts` в `child.ts`, что привело бы к циклу импортов:

```text

parent.ts -> child.ts -> parent.ts

```

## Удалённое определение области видимости {#remote-scoping}

При использовании NgModule, чтобы избежать добавления импортов, создающих циклы, дополнительный код добавляется в класс `NgModule`, где объявляется компонент, связывающий зависимости.

Это известно как «удалённое определение области видимости».

## Библиотеки {#libraries}

К сожалению, код «удалённого определения области видимости» имеет побочные эффекты — что мешает tree-shaking — и не может использоваться в библиотеках.
Поэтому при сборке библиотек с настройкой `"compilationMode": "partial"` любой компонент, для которого потребовался бы циклический импорт, вызовет эту ошибку компилятора `NG3003`.

## Отладка ошибки {#debugging-the-error}

Цикл, который был бы сгенерирован, отображается в составе сообщения об ошибке.
Например:

```text {hideCopy}

The component Child is used in the template but importing it would create a cycle:
/parent.ts -> /child.ts -> /parent.ts

```

Используйте это, чтобы понять, как ссылаемый компонент, Pipe или директива имеет зависимость обратно к компонируемому компоненту.
Вот несколько способов исправить проблему:

- Попробуйте переорганизовать зависимости, чтобы избежать цикла.
  Например, используйте промежуточный интерфейс, хранящийся в независимом файле, который можно импортировать в оба зависимых файла без создания цикла импортов.
- Переместите классы, ссылающиеся друг на друга, в один файл, чтобы избежать импортов между ними.
- Преобразуйте операторы импорта в импорты только типов \(используя синтаксис `import type`\), если импортируемые объявления используются только как типы, так как импорты только типов не способствуют циклам.
