# Структура файлов рабочего пространства и проекта {#workspace-and-project-file-structure}

Вы разрабатываете приложения в контексте рабочего пространства Angular.
Рабочее пространство содержит файлы одного или нескольких проектов.
Проект — это набор файлов, составляющих приложение или разделяемую библиотеку.

Команда Angular CLI `ng new` создаёт рабочее пространство.

```shell
ng new my-project
```

При выполнении этой команды CLI устанавливает необходимые npm-пакеты Angular и другие зависимости в новое рабочее пространство с корневым приложением, названным _my-project_.

По умолчанию `ng new` создаёт исходный скелет приложения на корневом уровне рабочего пространства вместе с его сквозными тестами.
Скелет представляет собой простое приветственное приложение, готовое к запуску и удобное для изменения.
Корневое приложение имеет то же имя, что и рабочее пространство, а исходные файлы находятся в подпапке `src/` рабочего пространства.

Это поведение по умолчанию подходит для типичного стиля разработки «multi-repo», где каждое приложение находится в своём собственном рабочем пространстве.
Начинающим и пользователям среднего уровня рекомендуется использовать `ng new` для создания отдельного рабочего пространства для каждого приложения.

Angular также поддерживает рабочие пространства с [несколькими проектами](#multiple-projects).
Такой тип среды разработки подходит для опытных пользователей, разрабатывающих разделяемые библиотеки,
и для предприятий, использующих стиль разработки «monorepo» с единым репозиторием и глобальной конфигурацией для всех проектов Angular.

Для настройки монорепо-рабочего пространства следует пропустить создание корневого приложения.
См. раздел [Настройка для многопроектного рабочего пространства](#multiple-projects) ниже.

## Файлы конфигурации рабочего пространства {#workspace-configuration-files}

Все проекты в рабочем пространстве разделяют одну [конфигурацию](reference/configs/workspace-config).
На верхнем уровне рабочего пространства находятся файлы конфигурации рабочего пространства, файлы конфигурации корневого приложения, а также подпапки с исходными файлами и тестами корневого приложения.

| Файлы конфигурации рабочего пространства | Назначение                                                                                                                                                                                                                                                                                                             |
| :--------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `.editorconfig`                          | Конфигурация для редакторов кода. См. [EditorConfig](https://editorconfig.org).                                                                                                                                                                                                                                        |
| `.gitignore`                             | Указывает файлы, специально не отслеживаемые [Git](https://git-scm.com).                                                                                                                                                                                                                                               |
| `README.md`                              | Документация рабочего пространства.                                                                                                                                                                                                                                                                                    |
| `angular.json`                           | Конфигурация CLI для всех проектов в рабочем пространстве, включая параметры конфигурации сборки, обслуживания и тестирования каждого проекта. Подробнее см. в разделе [Конфигурация рабочего пространства Angular](reference/configs/workspace-config).                                                                |
| `package.json`                           | Настраивает [npm-зависимости](reference/configs/npm-packages), доступные для всех проектов в рабочем пространстве. Формат и содержимое этого файла см. в [документации npm](https://docs.npmjs.com/files/package.json).                                                                                                |
| `package-lock.json`                      | Предоставляет информацию о версиях всех пакетов, установленных в `node_modules` npm-клиентом. Подробнее см. в [документации npm](https://docs.npmjs.com/files/package-lock.json).                                                                                                                                      |
| `src/`                                   | Исходные файлы корневого проекта приложения.                                                                                                                                                                                                                                                                           |
| `public/`                                | Содержит изображения и другие файлы ресурсов для статической раздачи сервером разработки и копирования без изменений при сборке приложения.                                                                                                                                                                             |
| `node_modules/`                          | Установленные [npm-пакеты](reference/configs/npm-packages) для всего рабочего пространства. Зависимости `node_modules` на уровне рабочего пространства видны всем проектам.                                                                                                                                            |
| `tsconfig.json`                          | Базовая [конфигурация TypeScript](https://www.typescriptlang.org) для проектов в рабочем пространстве. Все остальные файлы конфигурации наследуются от этого базового файла. Подробнее см. в [соответствующей документации TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html#tsconfig-bases). |

## Файлы проекта приложения {#application-project-files}

По умолчанию команда CLI `ng new my-app` создаёт папку рабочего пространства с именем "my-app" и генерирует новый скелет приложения в папке `src/` на верхнем уровне рабочего пространства.
Вновь созданное приложение содержит исходные файлы для корневого модуля с корневым компонентом и шаблоном.

Когда структура файлов рабочего пространства готова, для добавления функциональности и данных в приложение можно использовать команду `ng generate` в командной строке.
Это исходное корневое приложение является _приложением по умолчанию_ для команд CLI (если вы не измените приложение по умолчанию после создания [дополнительных приложений](#multiple-projects)).

Для однопроектного рабочего пространства подпапка `src` рабочего пространства содержит исходные файлы (логику приложения, данные и ресурсы) для корневого приложения.
Для многопроектного рабочего пространства дополнительные проекты в папке `projects` содержат подпапку `project-name/src/` с той же структурой.

### Исходные файлы приложения {#application-source-files}

Файлы на верхнем уровне `src/` обеспечивают запуск приложения.
Подпапки содержат исходный код приложения и специфичную для него конфигурацию.

| Вспомогательные файлы приложения | Назначение                                                                                                                                                                                                                              |
| :------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `app/`                           | Содержит файлы компонентов, в которых определены логика и данные приложения. Подробности см. ниже.                                                                                                                                       |
| `favicon.ico`                    | Иконка для приложения в строке закладок.                                                                                                                                                                                                 |
| `index.html`                     | Главная HTML-страница, отображаемая при посещении сайта. CLI автоматически добавляет все файлы JavaScript и CSS при сборке приложения, поэтому обычно не нужно вручную добавлять теги `<script>` или `<link>`.                           |
| `main.ts`                        | Главная точка входа для приложения.                                                                                                                                                                                                      |
| `styles.css`                     | Глобальные CSS-стили, применяемые ко всему приложению.                                                                                                                                                                                   |

Внутри папки `src` папка `app` содержит логику и данные проекта.
Здесь размещаются компоненты Angular, шаблоны и стили.

| Файлы `src/app/`        | Назначение                                                                                                                                                                                                                                                                                              |
| :---------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `app.config.ts`         | Определяет конфигурацию приложения, указывающую Angular, как собрать приложение. По мере добавления новых провайдеров в приложение их следует объявлять здесь.<br><br>_Генерируется только при использовании параметра `--standalone`._                                                                   |
| `app.component.ts`      | Определяет корневой компонент приложения с именем `AppComponent`. Представление, связанное с этим корневым компонентом, становится корнем иерархии представлений по мере добавления компонентов и сервисов в приложение.                                                                                  |
| `app.component.html`    | Определяет HTML-шаблон, связанный с `AppComponent`.                                                                                                                                                                                                                                                     |
| `app.component.css`     | Определяет CSS-таблицу стилей для `AppComponent`.                                                                                                                                                                                                                                                       |
| `app.component.spec.ts` | Определяет модульный тест для `AppComponent`.                                                                                                                                                                                                                                                            |
| `app.module.ts`         | Определяет корневой модуль с именем `AppModule`, указывающий Angular, как собрать приложение. Изначально объявляет только `AppComponent`. По мере добавления компонентов в приложение их необходимо объявлять здесь.<br><br>_Генерируется только при использовании параметра `--standalone false`._       |
| `app.routes.ts`         | Определяет конфигурацию маршрутизации приложения.                                                                                                                                                                                                                                                       |

### Файлы конфигурации приложения {#application-configuration-files}

Специфичные для приложения файлы конфигурации корневого приложения находятся на корневом уровне рабочего пространства.
Для многопроектного рабочего пространства специфичные для проекта файлы конфигурации находятся в корне проекта, в директории `projects/project-name/`.

Специфичные для проекта файлы конфигурации [TypeScript](https://www.typescriptlang.org) наследуются от общего для рабочего пространства `tsconfig.json`.

| Специфичные для приложения файлы конфигурации | Назначение                                                                                                                                                                          |
| :---------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `tsconfig.app.json`                             | Специфичная для приложения [конфигурация TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html), включая [параметры компилятора Angular](reference/configs/angular-compiler-options). |
| `tsconfig.spec.json`                            | [Конфигурация TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html) для тестов приложения.                                                                  |

## Несколько проектов {#multiple-projects}

Многопроектное рабочее пространство подходит для организации, использующей единый репозиторий и глобальную конфигурацию для нескольких проектов Angular (модель «monorepo»).
Многопроектное рабочее пространство также поддерживает разработку библиотек.

### Настройка для многопроектного рабочего пространства {#setting-up-for-a-multi-project-workspace}

Если вы планируете иметь несколько проектов в рабочем пространстве, вы можете пропустить создание исходного приложения при создании рабочего пространства и дать рабочему пространству уникальное имя.
Следующая команда создаёт рабочее пространство со всеми файлами конфигурации рабочего пространства, но без корневого приложения.

```shell
ng new my-workspace --no-create-application
```

После этого можно генерировать приложения и библиотеки с именами, уникальными в рамках рабочего пространства.

```shell
cd my-workspace
ng generate application my-app
ng generate library my-lib
```

### Структура файлов нескольких проектов {#multiple-project-file-structure}

Первое явно созданное приложение помещается в папку `projects` вместе со всеми остальными проектами в рабочем пространстве.
Вновь созданные библиотеки также добавляются в директорию `projects`.
При создании проектов таким способом структура файлов рабочего пространства полностью соответствует структуре [файла конфигурации рабочего пространства](reference/configs/workspace-config), `angular.json`.

```markdown
my-workspace/
├── … (файлы конфигурации рабочего пространства)
└── projects/ (приложения и библиотеки)
├── my-app/ (явно созданное приложение)
│ └── … (специфичный для приложения код и конфигурация)
└── my-lib/ (созданная библиотека)
└── … (специфичный для библиотеки код и конфигурация)
```

## Файлы библиотечного проекта {#library-project-files}

При генерации библиотеки с помощью CLI (например, командой `ng generate library my-lib`) созданные файлы помещаются в папку `projects/` рабочего пространства.
Подробнее о создании собственных библиотек см. в разделе [Создание библиотек](tools/libraries/creating-libraries).

В отличие от приложения, библиотека имеет собственный файл конфигурации `package.json`.

В папке `projects/` папка `my-lib` содержит код библиотеки.

| Исходные файлы библиотеки | Назначение                                                                                                                                                                                        |
| :------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `src/lib`                 | Содержит логику и данные библиотечного проекта. Как и проект приложения, библиотечный проект может содержать компоненты, сервисы, модули, директивы и Pipe.                                       |
| `src/public-api.ts`       | Указывает все файлы, экспортируемые из библиотеки.                                                                                                                                                |
| `ng-package.json`         | Файл конфигурации, используемый [ng-packagr](https://github.com/ng-packagr/ng-packagr) для сборки библиотеки.                                                                                     |
| `package.json`            | Настраивает [npm-зависимости](reference/configs/npm-packages), необходимые для этой библиотеки.                                                                                                   |
| `tsconfig.lib.json`       | Специфичная для библиотеки [конфигурация TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html), включая [параметры компилятора Angular](reference/configs/angular-compiler-options). |
| `tsconfig.lib.prod.json`  | Специфичная для библиотеки [конфигурация TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html), используемая при сборке библиотеки в режиме продакшена.                    |
| `tsconfig.spec.json`      | [Конфигурация TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html) для модульных тестов библиотеки.                                                                       |
