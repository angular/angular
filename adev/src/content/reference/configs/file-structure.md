# Структура файлов рабочего пространства и проекта

Разработка приложений ведется в контексте рабочего пространства (workspace) Angular. Рабочее пространство содержит
файлы одного или нескольких проектов. Проект — это набор файлов, из которых состоит приложение или публикуемая
библиотека.

Команда Angular CLI `ng new` создает рабочее пространство.

```shell
ng new my-project
```

При выполнении этой команды CLI устанавливает необходимые npm-пакеты Angular и другие зависимости в новое рабочее
пространство с корневым приложением с именем _my-project_.

По умолчанию `ng new` создает начальный скелет приложения на корневом уровне рабочего пространства вместе с его
end-to-end тестами. Скелет представляет собой простое приветственное приложение, готовое к запуску и простое в
изменении. Корневое приложение имеет то же имя, что и рабочее пространство, а исходные файлы находятся в подпапке
`src/` рабочего пространства.

Это поведение по умолчанию подходит для типичного стиля разработки "multi-repo", где каждое приложение находится в
собственном рабочем пространстве. Начинающим и разработчикам среднего уровня рекомендуется использовать `ng new` для
создания отдельного рабочего пространства для каждого приложения.

Angular также поддерживает рабочие пространства с [несколькими проектами](#multiple-projects). Этот тип среды
разработки подходит для опытных пользователей, разрабатывающих публикуемые библиотеки, а также для предприятий,
использующих стиль разработки "monorepo" с единым репозиторием и глобальной конфигурацией для всех проектов Angular.

Чтобы настроить монорепозиторное рабочее пространство, необходимо пропустить создание корневого приложения. См. раздел
[Настройка для рабочего пространства с несколькими проектами](#multiple-projects) ниже.

## Файлы конфигурации рабочего пространства {#workspace-configuration-files}

Все проекты в рабочем пространстве используют общую
[конфигурацию](reference/configs/workspace-config). На верхнем уровне рабочего пространства находятся файлы
конфигурации для всего рабочего пространства, файлы конфигурации для корневого приложения, а также подпапки для
исходных файлов и файлов тестов корневого приложения.

| Файлы конфигурации рабочего пространства | Назначение                                                                                                                                                                                                                                                                                                                    |
| :---------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `.editorconfig`                           | Конфигурация для редакторов кода. См. [EditorConfig](https://editorconfig.org).                                                                                                                                                                                                                                               |
| `.gitignore`                              | Указывает намеренно неотслеживаемые файлы, которые [Git](https://git-scm.com) должен игнорировать.                                                                                                                                                                                                                            |
| `README.md`                               | Документация рабочего пространства.                                                                                                                                                                                                                                                                                           |
| `angular.json`                            | Конфигурация CLI для всех проектов в рабочем пространстве, включая параметры конфигурации для сборки, обслуживания и тестирования каждого проекта. Подробнее см. в разделе [Конфигурация рабочего пространства Angular](reference/configs/workspace-config).                                                                   |
| `package.json`                            | Настраивает [зависимости npm-пакетов](reference/configs/npm-packages), доступные для всех проектов в рабочем пространстве. Формат и содержимое этого файла описаны в [документации npm](https://docs.npmjs.com/files/package.json).                                                                                            |
| `package-lock.json`                       | Содержит информацию о версиях всех пакетов, установленных в `node_modules` клиентом npm. Подробнее см. в [документации npm](https://docs.npmjs.com/files/package-lock.json).                                                                                                                                                   |
| `src/`                                    | Исходные файлы для корневого проекта приложения.                                                                                                                                                                                                                                                                              |
| `public/`                                 | Содержит изображения и другие файлы ресурсов, которые будут обслуживаться как статические файлы dev-сервером и копироваться как есть при сборке приложения.                                                                                                                                                                    |
| `node_modules/`                           | Установленные [npm-пакеты](reference/configs/npm-packages) для всего рабочего пространства. Зависимости `node_modules` уровня рабочего пространства видны всем проектам.                                                                                                                                                      |
| `tsconfig.json`                           | Базовая конфигурация [TypeScript](https://www.typescriptlang.org) для проектов в рабочем пространстве. Все остальные файлы конфигурации наследуются от этого базового файла. Подробнее см. в [соответствующей документации TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html#tsconfig-bases). |

## Файлы проекта приложения {#application-project-files}

По умолчанию команда CLI `ng new my-app` создает папку рабочего пространства с именем "my-app" и генерирует новый
скелет приложения в папке `src/` на верхнем уровне рабочего пространства. Вновь созданное приложение содержит исходные
файлы для корневого модуля с корневым компонентом и шаблоном.

После того как структура файлов рабочего пространства готова, вы можете использовать команду `ng generate` в командной
строке для добавления функциональности и данных в приложение. Это начальное корневое приложение является _приложением
по умолчанию_ для команд CLI (пока вы не измените умолчание после создания
[дополнительных приложений](#multiple-projects)).

Для рабочего пространства с одним приложением подпапка `src` содержит исходные файлы (логику приложения, данные и
ресурсы) для корневого приложения. Для рабочего пространства с несколькими проектами дополнительные проекты в папке
`projects` содержат подпапку `project-name/src/` с аналогичной структурой.

### Исходные файлы приложения {#application-source-files}

Файлы на верхнем уровне `src/` поддерживают запуск вашего приложения. Подпапки содержат исходный код приложения и
специфичную для приложения конфигурацию.

| Вспомогательные файлы приложения | Назначение                                                                                                                                                                                                                                  |
| :-------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `app/`                            | Содержит файлы компонентов, в которых определены логика и данные приложения. Подробнее см. ниже.                                                                                                                                            |
| `favicon.ico`                     | Иконка для этого приложения на панели закладок.                                                                                                                                                                                             |
| `index.html`                      | Главная HTML-страница, отображаемая при посещении сайта. CLI автоматически добавляет все файлы JavaScript и CSS при сборке приложения, поэтому обычно не нужно вручную добавлять теги `<script>` или `<link>`.                               |
| `main.ts`                         | Главная точка входа в приложение.                                                                                                                                                                                                           |
| `styles.css`                      | Глобальные CSS-стили, применяемые ко всему приложению.                                                                                                                                                                                      |

Внутри папки `src` папка `app` содержит логику и данные вашего проекта. Компоненты, шаблоны и стили Angular находятся
здесь.

| Файлы `src/app/`        | Назначение                                                                                                                                                                                                                                                                                              |
| :---------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `app.config.ts`         | Определяет конфигурацию приложения, сообщающую Angular, как собрать приложение. По мере добавления провайдеров в приложение их следует объявлять здесь.<br><br>_Генерируется только при использовании параметра `--standalone`._                                                                          |
| `app.component.ts`      | Определяет корневой компонент приложения с именем `AppComponent`. Представление, связанное с этим корневым компонентом, становится корнем иерархии представлений по мере добавления компонентов и сервисов в приложение.                                                                                 |
| `app.component.html`    | Определяет HTML-шаблон, связанный с `AppComponent`.                                                                                                                                                                                                                                                     |
| `app.component.css`     | Определяет CSS-таблицу стилей для `AppComponent`.                                                                                                                                                                                                                                                       |
| `app.component.spec.ts` | Определяет модульный тест для `AppComponent`.                                                                                                                                                                                                                                                           |
| `app.module.ts`         | Определяет корневой модуль с именем `AppModule`, сообщающий Angular, как собрать приложение. Изначально объявляет только `AppComponent`. По мере добавления компонентов в приложение их необходимо объявлять здесь.<br><br>_Генерируется только при использовании параметра `--standalone false`._ |
| `app.routes.ts`         | Определяет конфигурацию маршрутизации приложения.                                                                                                                                                                                                                                                       |

### Файлы конфигурации приложения {#application-configuration-files}

Специфичные для приложения файлы конфигурации для корневого приложения находятся на корневом уровне рабочего
пространства. Для рабочего пространства с несколькими проектами специфичные для проекта файлы конфигурации находятся в
корне проекта, в папке `projects/project-name/`.

Специфичные для проекта файлы конфигурации [TypeScript](https://www.typescriptlang.org) наследуются от общего для
рабочего пространства `tsconfig.json`.

| Специфичные для приложения файлы конфигурации | Назначение                                                                                                                                                                                                                   |
| :--------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `tsconfig.app.json`                            | Специфичная для приложения [конфигурация TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html), включая [параметры компилятора Angular](reference/configs/angular-compiler-options).                   |
| `tsconfig.spec.json`                           | [Конфигурация TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html) для тестов приложения.                                                                                                             |

## Несколько проектов {#multiple-projects}

Рабочее пространство с несколькими проектами подходит для организаций, использующих единый репозиторий и глобальную
конфигурацию для нескольких проектов Angular (модель "monorepo"). Рабочее пространство с несколькими проектами также
поддерживает разработку библиотек.

### Настройка рабочего пространства с несколькими проектами {#setting-up-for-a-multi-project-workspace}

Если вы планируете иметь несколько проектов в рабочем пространстве, можно пропустить начальное создание приложения при
создании рабочего пространства и дать ему уникальное имя. Следующая команда создает рабочее пространство со всеми
файлами конфигурации уровня рабочего пространства, но без корневого приложения.

```shell
ng new my-workspace --no-create-application
```

Затем можно создавать приложения и библиотеки с именами, уникальными в пределах рабочего пространства.

```shell
cd my-workspace
ng generate application my-app
ng generate library my-lib
```

### Структура файлов при нескольких проектах {#multiple-project-file-structure}

Первое явно созданное приложение помещается в папку `projects` вместе со всеми остальными проектами рабочего
пространства. Вновь созданные библиотеки также добавляются в папку `projects`. При создании проектов таким образом
структура файлов рабочего пространства полностью соответствует структуре
[файла конфигурации рабочего пространства](reference/configs/workspace-config), `angular.json`.

```markdown
my-workspace/
├── … (файлы конфигурации уровня рабочего пространства)
└── projects/ (приложения и библиотеки)
├── my-app/ (явно созданное приложение)
│ └── … (специфичный для приложения код и конфигурация)
└── my-lib/ (созданная библиотека)
└── … (специфичный для библиотеки код и конфигурация)
```

## Файлы библиотечного проекта {#library-project-files}

При создании библиотеки с помощью CLI (командой `ng generate library my-lib`) созданные файлы помещаются в папку
`projects/` рабочего пространства. Подробнее о создании собственных библиотек см. в разделе
[Создание библиотек](tools/libraries/creating-libraries).

В отличие от приложения, библиотека имеет собственный файл конфигурации `package.json`.

В папке `projects/` папка `my-lib` содержит код вашей библиотеки.

| Исходные файлы библиотеки | Назначение                                                                                                                                                                                                                                        |
| :------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/lib`                 | Содержит логику и данные библиотечного проекта. Как и проект приложения, библиотечный проект может содержать компоненты, сервисы, модули, директивы и pipe.                                                                                        |
| `src/public-api.ts`       | Указывает все файлы, экспортируемые из вашей библиотеки.                                                                                                                                                                                          |
| `ng-package.json`         | Файл конфигурации, используемый [ng-packagr](https://github.com/ng-packagr/ng-packagr) для сборки библиотеки.                                                                                                                                      |
| `package.json`            | Настраивает [зависимости npm-пакетов](reference/configs/npm-packages), необходимые для этой библиотеки.                                                                                                                                            |
| `tsconfig.lib.json`       | Специфичная для библиотеки [конфигурация TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html), включая [параметры компилятора Angular](reference/configs/angular-compiler-options).                                        |
| `tsconfig.lib.prod.json`  | Специфичная для библиотеки [конфигурация TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html), используемая при сборке библиотеки в production-режиме.                                                                     |
| `tsconfig.spec.json`      | [Конфигурация TypeScript](https://www.typescriptlang.org/docs/handbook/tsconfig-json.html) для модульных тестов библиотеки.                                                                                                                        |
