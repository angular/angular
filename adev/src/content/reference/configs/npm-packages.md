# npm-зависимости рабочего пространства {#workspace-npm-dependencies}

Фреймворк Angular, Angular CLI и компоненты, используемые приложениями Angular, упакованы как [npm-пакеты](https://docs.npmjs.com/getting-started/what-is-npm 'Что такое npm?') и распространяются через [реестр npm](https://docs.npmjs.com).

Вы можете загрузить и установить эти npm-пакеты с помощью [npm CLI-клиента](https://docs.npmjs.com/cli/install).
По умолчанию Angular CLI использует npm-клиент.

ПОДСКАЗКА: Информацию о необходимых версиях и установке `Node.js` и `npm` см. в разделе [Настройка локального окружения](tools/cli/setup-local 'Настройка для локальной разработки').

Если у вас уже есть проекты, запущенные на вашем компьютере и использующие другие версии Node.js и npm, рассмотрите возможность использования [nvm](https://github.com/creationix/nvm) для управления несколькими версиями Node.js и npm.

## `package.json` {#packagejson}

`npm` устанавливает пакеты, указанные в файле [`package.json`](https://docs.npmjs.com/files/package.json).

Команда CLI `ng new` создаёт файл `package.json` при создании нового рабочего пространства.
Этот `package.json` используется всеми проектами в рабочем пространстве, включая исходный проект приложения, создаваемый CLI при создании рабочего пространства.
Библиотеки, созданные с помощью `ng generate library`, будут иметь собственный файл `package.json`.

Изначально этот `package.json` включает _стартовый набор пакетов_, некоторые из которых требуются Angular, а другие поддерживают типичные сценарии приложения.
Вы добавляете пакеты в `package.json` по мере развития приложения.

## Зависимости по умолчанию {#default-dependencies}

Полный список пакетов Angular см. в [справочнике API](api).
