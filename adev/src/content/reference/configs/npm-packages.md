# npm-зависимости рабочего пространства

Фреймворк Angular, Angular CLI и компоненты, используемые приложениями Angular, упакованы как
[npm-пакеты](https://docs.npmjs.com/getting-started/what-is-npm 'Что такое npm?') и распространяются через
[реестр npm](https://docs.npmjs.com).

Вы можете загружать и устанавливать эти npm-пакеты с помощью
[npm CLI клиента](https://docs.npmjs.com/cli/install). По умолчанию Angular CLI использует клиент npm.

HELPFUL: Информацию о требуемых версиях и установке `Node.js` и `npm` см. в разделе
[Настройка локальной среды](tools/cli/setup-local 'Настройка для локальной разработки').

Если на вашем компьютере уже выполняются проекты, использующие другие версии Node.js и npm, рассмотрите возможность
использования [nvm](https://github.com/creationix/nvm) для управления несколькими версиями Node.js и npm.

## `package.json` {#packagejson}

`npm` устанавливает пакеты, указанные в файле
[`package.json`](https://docs.npmjs.com/files/package.json).

Команда CLI `ng new` создает файл `package.json` при создании нового рабочего пространства. Этот `package.json`
используется всеми проектами в рабочем пространстве, включая начальный проект приложения, создаваемый CLI при
создании рабочего пространства. Библиотеки, созданные с помощью `ng generate library`, будут включать собственный
файл `package.json`.

Изначально этот `package.json` включает _стартовый набор пакетов_, часть из которых необходима Angular, а другие
поддерживают распространенные сценарии приложений. Вы добавляете пакеты в `package.json` по мере развития приложения.

## Зависимости по умолчанию {#default-dependencies}

пространства Angular. Полный список пакетов Angular см. в [справочнике API](api).

Подробнее см. в [руководстве по устаревшим анимациям](guide/legacy-animations).                         |
| [`@angular/common`](api#common)                                                                | Часто используемые сервисы, pipe и директивы, предоставляемые командой Angular.                                                                                                                                                                   |
| `@angular/compiler`                                                                            | Компилятор шаблонов Angular. Понимает шаблоны Angular и может преобразовывать их в код, обеспечивающий работу приложения.                                                                                                                          |
| `@angular/compiler-cli`                                                                        | Компилятор Angular, вызываемый командами `ng build` и `ng serve` Angular CLI. Обрабатывает шаблоны Angular с помощью `@angular/compiler` в рамках стандартной компиляции TypeScript.                                                               |
