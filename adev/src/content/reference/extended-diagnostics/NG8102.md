# Nullish coalescing not nullable {#nullish-coalescing-not-nullable}

Эта диагностика обнаруживает бесполезный оператор нулевого слияния \(`??`\) в шаблонах Angular.
В частности, она ищет операции, где входные данные не являются «nullable», то есть их тип не включает `null` или `undefined`.
Для таких значений правая часть `??` никогда не будет использована.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Шаблон отображает `username`, если он присутствует, и 'root', если он
  // равен `null` или `undefined`.
  template: `<div>{{ username ?? 'root' }}</div>`,
})
class MyComponent {
  // `username` объявлен как `string`, который _не может_ быть `null` или
  // `undefined`.
  username: string = 'Angelino';
}
```

## В чём проблема? {#whats-wrong-with-that}

Использование оператора нулевого слияния с non-nullable входными данными не имеет эффекта и указывает на расхождение между допустимым типом значения и тем, как оно представлено в шаблоне.
Разработчик может разумно предположить, что правая часть оператора нулевого слияния отображается в некоторых случаях, но на самом деле она никогда не будет отображена.
Это может привести к путанице относительно ожидаемого результата программы.

## Что следует сделать вместо этого? {#what-should-i-do-instead}

Обновите шаблон и объявленный тип, чтобы они были согласованы.
Проверьте тип входных данных и убедитесь, действительно ли они должны быть nullable.

Если входные данные должны быть nullable, добавьте `null` или `undefined` к их типу, чтобы это обозначить.

```angular-ts
import {Component} from '@angular/core';

@Component({
  template: `<div>{{ username ?? 'root' }}</div>`,
})
class MyComponent {
  // `username` теперь nullable. Если он когда-либо будет установлен в `null`, будет
  // отображено 'root'.
  username: string | null = 'Angelino';
}
```

Если входные данные _не должны_ быть nullable, удалите оператор `??` и его правый операнд, поскольку TypeScript гарантирует, что значение всегда будет non-nullable.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Шаблон всегда отображает `username`, что гарантированно никогда не бывает `null`
  // или `undefined`.
  template: `<div>{{ username }}</div>`,
})
class MyComponent {
  username: string = 'Angelino';
}
```

## Требования к конфигурации {#configuration-requirements}

Для выдачи расширенных диагностик необходимо включить [`strictTemplates`](tools/cli/template-typecheck#strict-mode).
Для выдачи диагностики `nullishCoalescingNotNullable` также необходимо включить [`strictNullChecks`](tools/cli/template-typecheck#strict-null-checks).

## Что делать, если избежать этого невозможно? {#what-if-i-cant-avoid-this}

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "nullishCoalescingNotNullable": "suppress"
      }
    }
  }
}
```

Подробнее см. в разделе [настройка расширенных диагностик](extended-diagnostics#configuration).
