# Nullish coalescing not nullable {#nullish-coalescing-not-nullable}

Эта диагностика обнаруживает бесполезный оператор nullish coalescing (`??`) в Angular-шаблонах.
Конкретно, она ищет операции, где входное значение не является «nullable», то есть его тип не включает `null` или `undefined`.
Для таких значений правая часть `??` никогда не будет использована.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Шаблон отображает `username`, если он присутствует, и 'root', если
  // значение равно `null` или `undefined`.
  template: `<div>{{ username ?? 'root' }}</div>`,
})
class MyComponent {
  // `username` объявлен как `string`, который _не может_ быть `null` или
  // `undefined`.
  username: string = 'Angelino';
}
```

## В чём проблема? {#whats-wrong-with-that}

Использование оператора nullish coalescing с non-nullable входным значением не имеет никакого эффекта и указывает на расхождение между допустимым типом значения и тем, как оно представлено в шаблоне.
Разработчик может разумно предположить, что правая часть оператора nullish coalescing отображается в некоторых случаях, но на самом деле она никогда не будет отображена.
Это может приводить к путанице относительно ожидаемого вывода программы.

## Что следует сделать? {#what-should-i-do-instead}

Обновите шаблон и объявленный тип, чтобы привести их в соответствие.
Перепроверьте тип входного значения и убедитесь, что оно действительно должно быть nullable.

Если входное значение должно быть nullable, добавьте `null` или `undefined` к его типу, чтобы это отразить.

```angular-ts
import {Component} from '@angular/core';

@Component({
  template: `<div>{{ username ?? 'root' }}</div>`,
})
class MyComponent {
  // `username` теперь nullable. Если оно когда-либо будет установлено в `null`,
  // будет отображаться 'root'.
  username: string | null = 'Angelino';
}
```

Если входное значение _не_ должно быть nullable, удалите оператор `??` и его правый операнд, поскольку TypeScript гарантирует, что значение всегда будет non-nullable.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Шаблон всегда отображает `username`, которое гарантированно никогда не
  // будет `null` или `undefined`.
  template: `<div>{{ username }}</div>`,
})
class MyComponent {
  username: string = 'Angelino';
}
```

## Требования к конфигурации {#configuration-requirements}

Для работы любой расширенной диагностики необходимо включить [`strictTemplates`](tools/cli/template-typecheck#strict-mode).
Для генерации диагностики `nullishCoalescingNotNullable` также необходимо включить [`strictNullChecks`](tools/cli/template-typecheck#strict-null-checks).

## Что делать, если этого нельзя избежать? {#what-if-i-cant-avoid-this}

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "nullishCoalescingNotNullable": "suppress"
      }
    }
  }
}
```

См. [конфигурацию расширенной диагностики](extended-diagnostics#configuration) для получения дополнительной информации.
