# Недопустимый "банан в коробке"

Эта диагностика обнаруживает перевернутый синтаксис "банан в коробке" (banana-in-box)
для [двусторонней привязки](guide/templates/two-way-binding).

```html
<user-viewer ([user])="loggedInUser" />
```

## В чем проблема? {#what-is-the-problem}

В текущем виде `([var])` на самом деле является [привязкой события](guide/templates/event-listeners) с именем `[var]`.
Автор, вероятно, подразумевал двустороннюю привязку к переменной с именем `var`, но в написанном виде эта привязка не
имеет эффекта.

## Как исправить? {#what-should-i-do-instead}

Исправьте опечатку.
Как следует из названия, банан `(` должен находиться _внутри_ коробки `[]`.
В данном случае:

```html
<user-viewer [(user)]="loggedInUser" />
```

## Требования к конфигурации {#configuration-requirements}

Для работы любой расширенной диагностики необходимо включить [
`strictTemplates`](tools/cli/template-typecheck#strict-mode).
У `invalidBananaInBox` нет дополнительных требований, кроме `strictTemplates`.

## Что делать, если этого нельзя избежать? {#what-if-i-cant-avoid-this}

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "invalidBananaInBox": "suppress"
      }
    }
  }
}
```

Смотрите [конфигурацию расширенной диагностики](extended-diagnostics#configuration) для получения дополнительной
информации.
