# Optional chain not nullable {#optional-chain-not-nullable}

Эта диагностика обнаруживает случаи, когда левая часть операции optional chain (`.?`) не включает `null` или `undefined` в своём типе в Angular-шаблонах.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Безопасно выводит имя пользователя, даже если `user` равно `null` или `undefined`.
  template: `<div>User name: {{ user?.name }}</div>`,
})
class MyComponent {
  // `user` объявлен как объект, который _не может_ быть `null` или `undefined`.
  user: {name: string} = {name: 'Angelino'};
}
```

## В чём проблема? {#whats-wrong-with-that}

Использование оператора optional chain с non-nullable входным значением не имеет никакого эффекта и указывает на расхождение между допустимым типом значения и тем, как оно представлено в шаблоне.
Разработчик может разумно предположить, что результат оператора optional chain может быть `null` или `undefined`, но на самом деле он никогда не будет ни тем, ни другим.
Это может приводить к путанице относительно ожидаемого вывода программы.

## Что следует сделать? {#what-should-i-do-instead}

Обновите шаблон и объявленный тип, чтобы привести их в соответствие.
Перепроверьте тип входного значения и убедитесь, что оно действительно должно быть nullable.

Если входное значение должно быть nullable, добавьте `null` или `undefined` к его типу, чтобы это отразить.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Если `user` равно nullish, `name` не будет вычислено, и выражение вернёт
  // nullish-значение (`null` или `undefined`).
  template: `<div>{{ user?.name }}</div>`,
})
class MyComponent {
  user: {name: string} | null = {name: 'Angelino'};
}
```

Если входное значение не должно быть nullable, удалите оператор `?`.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Шаблон всегда отображает `name`, так как `user` гарантированно никогда не
  // будет `null` или `undefined`.
  template: `<div>{{ foo.bar }}</div>`,
})
class MyComponent {
  user: {name: string} = {name: 'Angelino'};
}
```

## Требования к конфигурации {#configuration-requirements}

Для работы любой расширенной диагностики необходимо включить [`strictTemplates`](tools/cli/template-typecheck#strict-mode).
Для генерации диагностики `optionalChainNotNullable` также необходимо включить [`strictNullChecks`](tools/cli/template-typecheck#strict-null-checks).

## Что делать, если этого нельзя избежать? {#what-if-i-cant-avoid-this}

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "optionalChainNotNullable": "suppress"
      }
    }
  }
}
```

См. [конфигурацию расширенной диагностики](extended-diagnostics#configuration) для получения дополнительной информации.
