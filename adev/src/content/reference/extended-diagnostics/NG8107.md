# Optional chain not nullable {#optional-chain-not-nullable}

Эта диагностика обнаруживает случаи, когда левая часть операции опциональной цепочки (`.?`) не включает `null` или `undefined` в своём типе в шаблонах Angular.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Безопасно выводим имя пользователя, даже если `user` равен `null` или `undefined`.
  template: `<div>User name: {{ user?.name }}</div>`,
})
class MyComponent {
  // `user` объявлен как объект, который _не может_ быть `null` или `undefined`.
  user: {name: string} = {name: 'Angelino'};
}
```

## В чём проблема? {#whats-wrong-with-that}

Использование оператора опциональной цепочки с non-nullable входными данными не имеет эффекта и указывает на расхождение между допустимым типом значения и тем, как оно представлено в шаблоне.
Разработчик может разумно предположить, что результат оператора опциональной цепочки может быть `null` или `undefined`, но на самом деле он никогда не будет ни тем, ни другим.
Это может привести к путанице относительно ожидаемого результата программы.

## Что следует сделать вместо этого? {#what-should-i-do-instead}

Обновите шаблон и объявленный тип, чтобы они были согласованы.
Проверьте тип входных данных и убедитесь, действительно ли они должны быть nullable.

Если входные данные должны быть nullable, добавьте `null` или `undefined` к их типу, чтобы это обозначить.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Если `user` равен null, `name` не будет вычислен, и выражение вернёт
  // null-значение (`null` или `undefined`).
  template: `<div>{{ user?.name }}</div>`,
})
class MyComponent {
  user: {name: string} | null = {name: 'Angelino'};
}
```

Если входные данные не должны быть nullable, удалите оператор `?`.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Шаблон всегда отображает `name`, так как `user` гарантированно никогда не бывает `null`
  // или `undefined`.
  template: `<div>{{ foo.bar }}</div>`,
})
class MyComponent {
  user: {name: string} = {name: 'Angelino'};
}
```

## Требования к конфигурации {#configuration-requirements}

Для выдачи расширенных диагностик необходимо включить [`strictTemplates`](tools/cli/template-typecheck#strict-mode).
Для выдачи диагностики `optionalChainNotNullable` также необходимо включить [`strictNullChecks`](tools/cli/template-typecheck#strict-null-checks).

## Что делать, если избежать этого невозможно? {#what-if-i-cant-avoid-this}

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "optionalChainNotNullable": "suppress"
      }
    }
  }
}
```

Подробнее см. в разделе [настройка расширенных диагностик](extended-diagnostics#configuration).
