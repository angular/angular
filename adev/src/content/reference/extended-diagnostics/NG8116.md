# Отсутствует структурная директива

Эта диагностика проверяет, что Standalone-компонент, использующий в шаблоне пользовательские структурные директивы (
например, `*select` или `*featureFlag`), импортировал их.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // Шаблон использует `*select`, но соответствующая директива не импортирована.
  imports: [],
  template: `<p *select="let data; from: source">{{ data }}</p>`,
})
class MyComponent {}
```

## В чем проблема? {#what-is-the-problem}

Использование структурной директивы без ее импорта приведет к ошибке во время выполнения, так как Angular попытается
выполнить привязку к свойству `select` HTML-элемента, которого не существует.

## Что следует сделать? {#what-should-i-do-instead}

Убедитесь, что соответствующая структурная директива импортирована в компонент:

```angular-ts
import {Component} from '@angular/core';
import {SelectDirective} from 'my-directives';

@Component({
  // Добавьте `SelectDirective` в массив `imports`, чтобы сделать ее доступной в шаблоне.
  imports: [SelectDirective],
  template: `<p *select="let data; from: source">{{ data }}</p>`,
})
class MyComponent {}
```

## Требования к конфигурации {#configuration-requirements}

Для срабатывания любой расширенной диагностики необходимо включить опцию [
`strictTemplates`](tools/cli/template-typecheck#strict-mode).
У `missingStructuralDirective` нет дополнительных требований, кроме `strictTemplates`.

## Что делать, если этого нельзя избежать? {#what-if-i-cant-avoid-this}

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "missingStructuralDirective": "suppress"
      }
    }
  }
}
```

Подробнее см. в разделе [конфигурация расширенной диагностики](extended-diagnostics#configuration).
