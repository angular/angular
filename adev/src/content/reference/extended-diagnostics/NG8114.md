# Оператор нулевого слияния без скобок

Эта диагностика выявляет случаи, когда оператор нулевого слияния (`??`) используется совместно с операторами логического
ИЛИ (`||`) или логического И (`&&`) без скобок, определяющих приоритет выполнения.

```angular-ts
import {Component, signal, Signal} from '@angular/core';

@Component({
  template: `<button [disabled]="hasPermission() && (task()?.disabled ?? true)">Run</button>`,
})
class MyComponent {
  hasPermission = input(false);
  task = input<Task | undefined>(undefined);
}
```

## Что здесь не так? {#what-is-the-problem}

Без скобок трудно понять, какой оператор вычисляется первым: `??` или `||`/`&&`. Это считается ошибкой в TypeScript и
JavaScript, но исторически допускалось в шаблонах Angular.

## Что следует сделать? {#what-should-i-do-instead}

В таких ситуациях всегда используйте скобки для явного указания приоритета. Если вы не уверены в изначальном замысле
кода, но хотите сохранить текущее поведение, оберните в скобки выражение с оператором `??`.

```angular-ts
import {Component, signal, Signal} from '@angular/core';

@Component({
  template: `<button [disabled]="hasPermission() && (task()?.disabled ?? true)">Run</button>`,
})
class MyComponent {
  hasPermission = input(false);
  task = input<Task | undefined>(undefined);
}
```

## Требования к конфигурации {#configuration-requirements}

Для работы любой расширенной диагностики необходимо включить опцию [
`strictTemplates`](tools/cli/template-typecheck#strict-mode).
У `unparenthesizedNullishCoalescing` нет дополнительных требований, кроме `strictTemplates`.

## Что делать, если этого нельзя избежать? {#what-if-i-cant-avoid-this}

Эту диагностику можно отключить, отредактировав файл `tsconfig.json` проекта:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "unparenthesizedNullishCoalescing": "suppress"
      }
    }
  }
}
```

Подробнее см. в разделе [конфигурация расширенной диагностики](extended-diagnostics#configuration).
