# Производительность {#performance}

Angular включает множество оптимизаций из коробки, но по мере роста приложений может потребоваться тонкая настройка как скорости загрузки, так и отзывчивости во время использования. В этих руководствах рассматриваются инструменты и техники Angular, помогающие создавать быстрые приложения.

## Производительность загрузки {#loading-performance}

Производительность загрузки определяет, насколько быстро приложение становится видимым и интерактивным. Медленная загрузка напрямую влияет на [Core Web Vitals](https://web.dev/vitals/), такие как Largest Contentful Paint (LCP) и Time to First Byte (TTFB).

| Техника                                                                                                 | Что делает                                                                                                                                                                                                                                                         | Когда использовать                                                                                    |
| :------------------------------------------------------------------------------------------------------ | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------------------------------------------- |
| [Ленивая загрузка маршрутов](best-practices/performance/lazy-loaded-routes#lazily-loaded-components-and-routes) | Откладывает загрузку компонентов маршрута до навигации, уменьшая размер начального бандла                                                                                                                                                                        | Приложения с несколькими маршрутами, не все из которых нужны при начальной загрузке                   |
| [Отложенная загрузка с `@defer`](best-practices/performance/defer)                                      | Разделяет компоненты на отдельные бандлы, загружаемые по требованию                                                                                                                                                                                               | Компоненты, не видимые при начальной отрисовке; тяжёлые сторонние библиотеки; контент ниже сгиба      |
| [Оптимизация изображений](best-practices/performance/image-optimization)                                | Приоритизирует LCP-изображения, лениво загружает остальные, генерирует адаптивные атрибуты `srcset`                                                                                                                                                               | Любое приложение, отображающее изображения                                                            |
| [Серверная отрисовка](best-practices/performance/ssr)                                                   | Отрисовывает страницы на сервере для более быстрой первой отрисовки и лучшего SEO, с [гидратацией](guide/hydration) для восстановления интерактивности и [инкрементной гидратацией](guide/incremental-hydration) для отложенной гидратации отдельных секций       | Контентоёмкие приложения, страницы, требующие индексации поисковыми системами                         |

## Производительность во время выполнения {#runtime-performance}

Производительность во время выполнения определяет отзывчивость приложения после загрузки. Система обнаружения изменений Angular поддерживает синхронизацию DOM с данными, а оптимизация того, как и когда она запускается, — основной рычаг улучшения производительности во время выполнения.

| Техника                                                          | Что делает                                                                                                  | Когда использовать                                                                               |
| :--------------------------------------------------------------- | :---------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------------- |
| [Zoneless обнаружение изменений](guide/zoneless)                 | Устраняет накладные расходы ZoneJS и запускает обнаружение изменений только при сигнале от сигналов или событий | Новые приложения (по умолчанию в Angular v21+) или существующие приложения, готовые к миграции   |
| [Медленные вычисления](best-practices/slow-computations)         | Выявляет и оптимизирует ресурсоёмкие выражения шаблона и хуки жизненного цикла                             | Профилирование выявляет конкретные компоненты, вызывающие медленные циклы обнаружения изменений  |
| [Пропуск поддеревьев компонентов](best-practices/skipping-subtrees) | Использует обнаружение изменений `OnPush` для пропуска неизменённых деревьев компонентов                   | Приложения, требующие более точного контроля над обнаружением изменений                          |
| [Zone-загрязнение](best-practices/zone-pollution)                | Предотвращает лишнее обнаружение изменений, вызванное сторонними библиотеками или таймерами                 | Zone-приложения, где профилирование выявляет чрезмерные циклы обнаружения изменений              |

## Измерение производительности {#measuring-performance}

Понять, что оптимизировать, не менее важно, чем знать, как это делать. Angular интегрируется с инструментами разработчика браузера, чтобы помочь найти узкие места.

| Инструмент                                                                         | Что делает                                                                                                                                                                                         |
| :--------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Профилирование в Chrome DevTools](best-practices/profiling-with-chrome-devtools) | Записывает специфичные для Angular данные о производительности вместе с профилированием браузера, с цветными flame-диаграммами, показывающими отрисовку компонентов, циклы обнаружения изменений и хуки жизненного цикла |
| [Angular DevTools](tools/devtools)                                                 | Расширение браузера, предоставляющее инспектор дерева компонентов и профайлер для визуализации циклов обнаружения изменений                                                                        |

## С чего начать оптимизацию {#what-to-optimize-first}

Если не знаете, с чего начать, сначала выполните профилирование приложения с помощью [Angular-трека Chrome DevTools](best-practices/profiling-with-chrome-devtools) для выявления конкретных узких мест.

В качестве общей отправной точки:

- **Медленная начальная загрузка** — используйте [`@defer`](best-practices/performance/defer) для выноса крупных компонентов из основного бандла, [`NgOptimizedImage`](best-practices/performance/image-optimization) для приоритизации изображений выше сгиба и [серверную отрисовку](best-practices/performance/ssr) для более быстрой доставки контента.
- **Медленные взаимодействия после загрузки** — проверьте, включено ли [Zoneless обнаружение изменений](guide/zoneless), поищите [медленные вычисления](best-practices/slow-computations) в шаблонах или хуках жизненного цикла и рассмотрите [`OnPush`](best-practices/skipping-subtrees) для сокращения лишнего обнаружения изменений.
