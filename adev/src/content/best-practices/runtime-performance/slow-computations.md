# Медленные вычисления {#slow-computations}

В каждом цикле обнаружения изменений Angular синхронно:

- Вычисляет все выражения шаблонов во всех компонентах, если не указано иное, на основе стратегии обнаружения каждого компонента
- Выполняет хуки жизненного цикла `ngDoCheck`, `ngAfterContentChecked`, `ngAfterViewChecked` и `ngOnChanges`.
  Одно медленное вычисление в шаблоне или хуке жизненного цикла может замедлить весь процесс обнаружения изменений, поскольку Angular выполняет вычисления последовательно.

## Выявление медленных вычислений {#identifying-slow-computations}

Тяжёлые вычисления можно выявить с помощью профайлера Angular DevTools. На временной шкале производительности нажмите на столбец для предварительного просмотра конкретного цикла обнаружения изменений. Откроется столбчатая диаграмма, показывающая, сколько времени фреймворк провёл в обнаружении изменений для каждого компонента. При нажатии на компонент можно увидеть, сколько времени Angular провёл при вычислении его шаблона и хуков жизненного цикла.

<img alt="Angular DevTools profiler preview showing slow computation" src="assets/images/best-practices/runtime-performance/slow-computations.png">

Например, на скриншоте выше выбран второй записанный цикл обнаружения изменений. Angular потратил более 573 мс на этот цикл, причём больше всего времени — в `EmployeeListComponent`. На панели деталей видно, что Angular потратил более 297 мс на вычисление шаблона `EmployeeListComponent`.

## Оптимизация медленных вычислений {#optimizing-slow-computations}

Вот несколько методов устранения медленных вычислений:

- **Оптимизация базового алгоритма**. Это рекомендуемый подход. Если удаётся ускорить алгоритм, вызывающий проблему, можно ускорить весь механизм обнаружения изменений.
- **Кэширование с помощью чистых pipe**. Можно перенести тяжёлое вычисление в чистый [pipe](guide/templates/pipes). Angular повторно вычисляет чистый pipe только при обнаружении изменения его входных данных по сравнению с предыдущим вызовом.
- **Использование мемоизации**. [Мемоизация](https://en.wikipedia.org/wiki/Memoization) — это техника, аналогичная чистым pipe, с той разницей, что чистые pipe сохраняют только последний результат вычисления, тогда как мемоизация может хранить несколько результатов.
- **Избегание перерисовки/перекомпоновки в хуках жизненного цикла**. Определённые [операции](https://web.dev/avoid-large-complex-layouts-and-layout-thrashing/) заставляют браузер синхронно пересчитывать компоновку страницы или перерисовывать её. Поскольку перекомпоновка и перерисовка как правило медленны, следует избегать их выполнения в каждом цикле обнаружения изменений.

Чистые pipe и мемоизация имеют разные компромиссы. Чистые pipe — встроенная концепция Angular в отличие от мемоизации, которая является общей практикой программной инженерии для кэширования результатов функций. Накладные расходы на память при мемоизации могут быть значительными, если тяжёлое вычисление вызывается часто с разными аргументами.
