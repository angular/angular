# Сборка Angular-приложений

Angular CLI-приложение или библиотеку можно собрать командой `ng build`.
Она скомпилирует TypeScript-код в JavaScript, а также оптимизирует, объединит в бандл и минифицирует вывод по необходимости.

`ng build` выполняет только builder для цели `build` в проекте по умолчанию, как указано в `angular.json`.
Angular CLI включает четыре builders, обычно используемые как цели `build`:

| Builder                                         | Назначение                                                                                                                                                                                                                                        |
| ----------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `@angular/build:application`                    | Собирает приложение с клиентским бандлом, Node-сервером и prerendered-маршрутами на этапе сборки с помощью [esbuild](https://esbuild.github.io/).                                                                                                  |
| `@angular-devkit/build-angular:browser-esbuild` | Собирает клиентское приложение для использования в браузере с помощью [esbuild](https://esbuild.github.io/). См. [документацию `browser-esbuild`](tools/cli/build-system-migration#manual-migration-to-the-compatibility-builder) для дополнительной информации. |
| `@angular-devkit/build-angular:browser`         | Собирает клиентское приложение для использования в браузере с помощью [webpack](https://webpack.js.org/).                                                                                                                                                |
| `@angular/build:ng-packagr`                     | Собирает Angular-библиотеку в соответствии с [Angular Package Format](tools/libraries/angular-package-format).                                                                                                                                        |

Приложения, сгенерированные `ng new`, по умолчанию используют `@angular/build:application`.
Библиотеки, сгенерированные `ng generate library`, по умолчанию используют `@angular/build:ng-packagr`.

Какой builder используется для конкретного проекта, можно определить, посмотрев цель `build` для этого проекта.

```json
{
  "projects": {
    "my-app": {
      "architect": {
        // `ng build` invokes the Architect target named `build`.
        "build": {
          "builder": "@angular/build:application",
          …
        },
        "serve": { … }
        "test": { … }
        …
      }
    }
  }
}
```

На этой странице обсуждаются использование и опции `@angular/build:application`.

## Выходной каталог {#output-directory}

Результат этого процесса сборки выводится в каталог (`dist/${PROJECT_NAME}` по умолчанию).

## Настройка size budgets {#configuring-size-budgets}

По мере роста функциональности приложений растёт и их размер.
CLI позволяет задать пороговые значения размера в конфигурации, чтобы части приложения оставались в определённых вами границах размера.

Определите границы размера в файле конфигурации CLI `angular.json` в секции `budgets` для каждого [настроенного окружения](tools/cli/environments).

```json
{
  …
  "configurations": {
    "production": {
      …
      "budgets": [
        {
          "type": "initial",
          "maximumWarning": "250kb",
          "maximumError": "500kb"
        },
      ]
    }
  }
}
```

Можно указать size budgets для всего приложения и для отдельных частей.
Каждая запись budget настраивает budget данного типа.
Например, budget `initial` измеряет JavaScript и CSS, используемые для bootstrap приложения, что соответствует значению `Initial Total` в сводке вывода сборки.
Значения размера указываются в следующих форматах:

| Значение размера      | Подробности                                                                     |
| :-------------- | :-------------------------------------------------------------------------- |
| `123` или `123b` | Размер в байтах.                                                              |
| `123kb`         | Размер в килобайтах.                                                          |
| `123mb`         | Размер в мегабайтах.                                                          |
| `12%`           | Процент размера относительно baseline. \(Недействительно для значений baseline.\) |

При настройке budget builder предупреждает или сообщает об ошибке, когда данная часть приложения достигает или превышает заданную вами граничную величину.

Каждая запись budget — это JSON-объект со следующими свойствами:

| Свойство       | Значение                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| :------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| type           | Тип budget. Один из: <table> <thead> <tr> <th> Значение </th> <th> Подробности </th> </tr> </thead> <tbody> <tr> <td> <code>bundle</code> </td> <td> Размер конкретного бандла. | minimumWarning | Минимальный порог для предупреждения относительно baseline.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |

| warning        | Порог для предупреждения относительно baseline (min и max).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |

Чтобы настроить budget для лениво загружаемого бандла, используйте `type: "bundle"` и задайте `name` равным имени этого бандла.

```json
{
  "budgets": [
    {
      "type": "bundle",
      "name": "admin",
      "maximumWarning": "250kb",
      "maximumError": "300kb"
    }
  ]
}
```

Поле `name` соответствует имени бандла, а не выданному имени файла, поэтому оно не использует шаблоны wildcard или регулярные выражения вроде `admin.*.js`.

Следующий пример показывает budget, использующий baseline:

```json
{
  "type": "bundle",
  "name": "main",
  "baseline": "200kb",
  "maximumWarning": "10%",
  "maximumError": "20%"
}
```

В этом примере builder предупреждает, когда бандл вырастает сверх `220kb`, и выдаёт ошибку, когда он вырастает сверх `240kb`.

## Настройка зависимостей CommonJS {#configuring-commonjs-dependencies}

Всегда предпочитайте нативные [ECMAScript modules](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Statements/import) (ESM) во всём приложении и его зависимостях.
ESM — полностью специфицированный веб-стандарт и возможность языка JavaScript с сильной поддержкой статического анализа. Это делает оптимизации бандла мощнее, чем у других форматов модулей.

Angular CLI также поддерживает импорт зависимостей [CommonJS](https://nodejs.org/api/modules.html) в проект и автоматически включает эти зависимости в бандл.
Однако модули CommonJS могут мешать bundlers и minifiers эффективно оптимизировать эти модули, что приводит к большему размеру бандла.
Дополнительную информацию см. в [How CommonJS is making your bundles larger](https://web.dev/commonjs-larger-bundles).

Angular CLI выводит предупреждения, если обнаруживает, что браузерное приложение зависит от модулей CommonJS.
При встрече с зависимостью CommonJS рассмотрите возможность попросить мейнтейнера поддержать ECMAScript modules, внести эту поддержку самостоятельно или использовать альтернативную зависимость, соответствующую вашим нуждам.
Если лучший вариант — использовать зависимость CommonJS, эти предупреждения можно отключить, добавив имя модуля CommonJS в опцию `allowedCommonJsDependencies` в опциях `build` в `angular.json`.

```json
"build": {
  "builder": "@angular/build:application",
  "options": {
     "allowedCommonJsDependencies": [
        "lodash"
     ]
     …
   }
   …
},
```

## Настройка совместимости с браузерами {#configuring-browser-compatibility}

Angular CLI использует [Browserslist](https://github.com/browserslist/browserslist), чтобы обеспечить совместимость с разными версиями браузеров.
В зависимости от поддерживаемых браузеров Angular автоматически трансформирует определённые возможности JavaScript и CSS, чтобы собранное приложение не использовало возможность, не реализованную поддерживаемым браузером. Однако Angular CLI не добавляет автоматически polyfills для отсутствующих Web API. Используйте опцию `polyfills` в `angular.json` для добавления polyfills.

По умолчанию Angular CLI использует конфигурацию `browserslist`, которая [соответствует браузерам, поддерживаемым Angular](reference/versions#browser-support) для текущей major-версии.

Чтобы переопределить внутреннюю конфигурацию, выполните [`ng generate config browserslist`](cli/generate/config), что генерирует файл конфигурации `.browserslistrc` в каталоге проекта, соответствующий поддерживаемым Angular браузерам.

См. [репозиторий browserslist](https://github.com/browserslist/browserslist) для дополнительных примеров нацеливания на конкретные браузеры и версии.
Избегайте расширения этого списка на большее число браузеров. Даже если код приложения совместим шире, сам Angular может таковым не быть.
Следует только _сокращать_ набор браузеров или версий в этом списке.

HELPFUL: Используйте [browsersl.ist](https://browsersl.ist) для отображения совместимых браузеров для запроса `browserslist`.

## Встраивание critical CSS {#critical-css-inlining}

Angular может встраивать определения critical CSS приложения для улучшения [First Contentful Paint (FCP)](https://web.dev/first-contentful-paint).
Эта опция включена по умолчанию. Встраивание можно отключить в [опциях кастомизации `styles`](reference/configs/workspace-config#styles-optimization-options).

Эта оптимизация извлекает CSS, необходимый для отрисовки начального viewport, и встраивает его напрямую в сгенерированный HTML, позволяя браузеру быстрее отображать контент без ожидания загрузки полных таблиц стилей. Остальной CSS затем загружается асинхронно в фоне. Angular CLI использует [Beasties](https://github.com/danielroe/beasties) для анализа HTML и стилей приложения.
