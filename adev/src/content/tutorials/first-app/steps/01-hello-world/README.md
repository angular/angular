# Hello world

Этот первый урок служит отправной точкой, от которой каждый последующий урок добавляет новые функции для создания полноценного приложения Angular. В этом уроке мы обновим приложение, чтобы оно отображало знаменитый текст «Hello World».

<docs-video src="https://www.youtube.com/embed/UnOwDuliqZA?si=uML-cDRbrxmYdD_9"/>

## Чему вы научитесь {#what-youll-learn}

Если приложение успешно запускается в браузере и отображает контент без ошибок, это подтверждает, что ваша среда разработки и IDE правильно настроены для начала разработки приложения Angular.

ПРИМЕЧАНИЕ: Если вы работаете со встроенным редактором, перейдите сразу к [шагу четыре](#create-hello-world).
При работе в браузерной среде вам не нужно выполнять `ng serve` для запуска приложения. Другие команды, например `ng generate`, можно выполнять в консоли справа.

<docs-workflow>

<docs-step title="Download the default app">
Начните с нажатия на иконку «Download» в правой верхней части редактора кода. Это загрузит `.zip`-файл с исходным кодом данного руководства. Откройте его в локальном терминале и IDE, затем перейдите к тестированию приложения по умолчанию.

На любом шаге руководства вы можете нажать эту иконку, чтобы загрузить исходный код текущего шага и начать с него.
</docs-step>

<docs-step title="Test the default app">
На этом шаге, после загрузки стартового приложения, вы соберёте приложение Angular по умолчанию.
Это подтвердит, что ваша среда разработки содержит всё необходимое для продолжения работы с руководством.

В панели **Terminal** вашей IDE:

1. В директории проекта перейдите в папку `first-app`.
1. Выполните эту команду для установки зависимостей, необходимых для запуска приложения.

   ```shell
   npm install
   ```

1. Выполните эту команду для сборки и запуска приложения по умолчанию.

   ```shell
   ng serve
   ```

   Приложение должно собраться без ошибок.

1. В веб-браузере на вашем компьютере разработки откройте `http://localhost:4200`.
1. Убедитесь, что в браузере отображается сайт по умолчанию.
1. Вы можете оставить `ng serve` работающим пока выполняете следующие шаги.
   </docs-step>

<docs-step title="Review the files in the project">
На этом шаге вы познакомитесь с файлами, из которых состоит приложение Angular по умолчанию.

В панели **Explorer** вашей IDE:

1. В директории проекта перейдите в папку `first-app`.
1. Откройте директорию `src`, чтобы увидеть эти файлы.
   1. В файловом менеджере найдите файлы приложения Angular (`/src`).
      1. `index.html` — HTML-шаблон верхнего уровня приложения.
      1. `styles.css` — таблица стилей верхнего уровня приложения.
      1. `main.ts` — точка запуска приложения.
      1. `favicon.ico` — иконка приложения, как и на любом другом сайте.
   1. В файловом менеджере найдите файлы компонента приложения Angular (`/app`).
      1. `app.ts` — исходный файл, описывающий компонент `app-root`.
         Это компонент верхнего уровня Angular в приложении. Компонент — это основной строительный блок приложения Angular.
         Описание компонента включает его код, HTML-шаблон и стили, которые могут быть описаны в этом файле или в отдельных файлах.

         В данном приложении стили находятся в отдельном файле, тогда как код компонента и HTML-шаблон — в этом файле.

      1. `app.css` — таблица стилей для этого компонента.
      1. Новые компоненты добавляются в эту директорию.

   1. В файловом менеджере найдите директорию изображений (`/assets`), содержащую изображения, используемые приложением.
   1. В файловом менеджере найдите файлы и директории, необходимые для сборки и запуска приложения Angular, но с которыми вы обычно не взаимодействуете напрямую.
      1. `.angular` содержит файлы, необходимые для сборки приложения Angular.
      1. `.e2e` содержит файлы для тестирования приложения.
      1. `.node_modules` содержит пакеты node.js, используемые приложением.
      1. `angular.json` описывает приложение Angular для инструментов сборки.
      1. `package.json` используется `npm` (менеджером пакетов node) для запуска готового приложения.
      1. `tsconfig.*` — файлы, описывающие конфигурацию приложения для компилятора TypeScript.

После изучения файлов проекта Angular переходите к следующему шагу.
</docs-step>
