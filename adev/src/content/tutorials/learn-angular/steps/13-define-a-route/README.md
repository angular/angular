# Определение маршрута {#define-a-route}

После настройки приложения для использования Angular Router необходимо определить маршруты.

NOTE: Подробнее об [определении базового маршрута в подробном руководстве](/guide/routing/define-routes).

В этом упражнении вы научитесь добавлять и настраивать маршруты в приложении.

<hr>

<docs-workflow>

<docs-step title="Определите маршрут в `app.routes.ts`">

В приложении есть две страницы для отображения: (1) главная страница и (2) страница пользователя.

Чтобы определить маршрут, добавьте объект маршрута в массив `routes` в файле `app.routes.ts`, который содержит:

- `path` маршрута (автоматически начинается с корневого пути, то есть `/`)
- `component`, который должен отображаться при этом маршруте

```ts
import {Routes} from '@angular/router';
import {Home} from './home/home';

export const routes: Routes = [
  {
    path: '',
    component: Home,
  },
];
```

Приведённый выше код является примером добавления `Home` в качестве маршрута. Реализуйте это вместе с `User` в редакторе.

Используйте `'user'` в качестве пути для `User`.

</docs-step>

<docs-step title="Добавьте заголовок в определение маршрута">

Помимо корректного определения маршрутов, Angular Router позволяет задавать заголовок страницы при навигации, добавляя свойство `title` в каждый маршрут.

В `app.routes.ts` добавьте свойство `title` в маршрут по умолчанию (`path: ''`) и маршрут `user`. Пример:

```ts {highlight:[7]}
import {Routes} from '@angular/router';
import {Home} from './home/home';

export const routes: Routes = [
  {
    path: '',
    title: 'App Home Page',
    component: Home,
  },
];
```

</docs-step>
