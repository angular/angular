# Сложные последовательности анимаций {#complex-animation-sequences}

ВАЖНО: Пакет `@angular/animations` теперь устарел (deprecated). Команда Angular рекомендует использовать нативный CSS с `animate.enter` и `animate.leave` для всего нового кода. Подробнее в новом [руководстве по анимациям](/guide/animations). Также см. [Миграция с пакета Angular Animations](guide/animations/migration), чтобы узнать, как начать переход на чистые CSS-анимации.

До сих пор мы изучали простые анимации отдельных HTML-элементов.
Angular также позволяет анимировать координированные последовательности, например целую сетку или список элементов при их входе на страницу и выходе с неё.
Можно запускать несколько анимаций параллельно или запускать отдельные анимации последовательно, одну за другой.

Функции, управляющие сложными последовательностями анимаций:

| Функции                           | Подробности                                                             |
| :-------------------------------- | :---------------------------------------------------------------------- |
| `query()`                         | Находит один или несколько внутренних HTML-элементов.                   |
| `stagger()`                       | Применяет каскадную задержку к анимациям нескольких элементов.          |
| [`group()`](api/animations/group) | Запускает несколько шагов анимации параллельно.                         |
| `sequence()`                      | Запускает шаги анимации один за другим.                                 |

## Функция query() {#the-query-function}

Большинство сложных анимаций полагаются на функцию `query()` для поиска дочерних элементов и применения к ним анимаций. Основные примеры:

| Примеры                                | Подробности                                                                                                                                                                                                            |
| :------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `query()` с последующим `animate()`    | Используется для запроса простых HTML-элементов и применения анимаций непосредственно к ним.                                                                                                                           |
| `query()` с последующим `animateChild()` | Используется для запроса дочерних элементов, к которым применены собственные метаданные анимации, и запуска таких анимаций \(которые иначе были бы заблокированы анимацией текущего/родительского элемента\). |

Первый аргумент `query()` — строка [CSS-селектора](https://developer.mozilla.org/docs/Web/CSS/CSS_Selectors), которая также может содержать следующие специфичные для Angular токены:

| Токены                     | Подробности                                           |
| :------------------------- | :---------------------------------------------------- |
| `:enter` <br /> `:leave`   | Для входящих/выходящих элементов.                     |
| `:animating`               | Для элементов, анимирующихся в данный момент.         |
| `@*` <br /> `@triggerName` | Для элементов с любым или конкретным триггером.       |
| `:self`                    | Сам анимирующийся элемент.                            |

<docs-callout title="Входящие и выходящие элементы">

Не все дочерние элементы фактически считаются входящими/выходящими; это иногда может быть неинтуитивным и запутывающим. Дополнительную информацию см. в [документации по API query](api/animations/query#entering-and-leaving-elements).

Также можно увидеть иллюстрацию этого в примере анимаций \(представленном в разделе [введение](guide/legacy-animations#about-this-guide)\) на вкладке «Querying».

</docs-callout>

## Анимация нескольких элементов с помощью функций query() и stagger() {#animate-multiple-elements-using-query-and-stagger-functions}

После запроса дочерних элементов через `query()` функция `stagger()` позволяет задать временной интервал между каждым запрошенным элементом, тем самым анимируя элементы с задержкой между ними.

В следующем примере показано, как использовать функции `query()` и `stagger()` для анимации списка \(героев\), добавляя каждого последовательно, с небольшой задержкой, сверху вниз.

- Используйте `query()` для поиска элемента, входящего на страницу и соответствующего определённым критериям.
- Для каждого из этих элементов используйте `style()` для установки одинакового начального стиля.
  Сделайте его прозрачным и используйте `transform` для смещения, чтобы он мог скользить на место.

- Используйте `stagger()` для задержки каждой анимации на 30 миллисекунд.
- Анимируйте каждый элемент на экране в течение 0.5 секунды, используя пользовательскую кривую сглаживания, одновременно делая его видимым и убирая трансформацию.

## Параллельная анимация с помощью функции group() {#parallel-animation-using-group-function}

Вы видели, как добавить задержку между последовательными анимациями.
Но иногда также нужно настроить параллельно выполняемые анимации.
Например, может потребоваться анимировать два CSS-свойства одного элемента, используя разные функции `easing` для каждого.
Для этого можно использовать функцию [`group()`](api/animations/group).

ПОЛЕЗНО: Функция [`group()`](api/animations/group) используется для группировки _шагов_ анимации, а не анимируемых элементов.

## Последовательные и параллельные анимации {#sequential-vs-parallel-animations}

В сложных анимациях одновременно может происходить много всего.
Но что, если нужно создать анимацию с несколькими шагами, выполняемыми один за другим? Ранее использовалась [`group()`](api/animations/group) для параллельного запуска нескольких анимаций.

Вторая функция `sequence()` позволяет запускать те же анимации последовательно.
Внутри `sequence()` шаги анимации состоят из вызовов функций `style()` или `animate()`.

- Используйте `style()` для немедленного применения предоставленных данных стиля.
- Используйте `animate()` для применения данных стиля за заданный временной интервал.

## Анимация элементов переупорядочиваемого списка {#animating-the-items-of-a-reordering-list}

Хотя Angular корректно анимирует элементы списка `*ngFor` из коробки, он не сможет этого сделать, если их порядок меняется.
Это объясняется тем, что Angular теряет связь с тем, какой элемент какой, что приводит к неработающим анимациям.
Единственный способ помочь Angular отслеживать такие элементы — присвоить `TrackByFunction` директиве `NgForOf`.
Это гарантирует, что Angular всегда знает, какой элемент какой, позволяя применять правильные анимации к правильным элементам всё время.

ВАЖНО: Если нужно анимировать элементы списка `*ngFor` и существует возможность изменения порядка этих элементов во время выполнения, всегда используйте `TrackByFunction`.

## Анимации и инкапсуляция представления компонента {#animations-and-component-view-encapsulation}

Анимации Angular основаны на структуре DOM компонентов и не учитывают напрямую [инкапсуляцию представления](guide/components/styling#style-scoping). Это означает, что компоненты с `ViewEncapsulation.Emulated` ведут себя точно так же, как при `ViewEncapsulation.None` (`ViewEncapsulation.ShadowDom` и `ViewEncapsulation.ExperimentalIsolatedShadowDom` ведут себя иначе, как будет обсуждено ниже).

Например, если функция `query()` применяется на верхнем уровне дерева компонентов с эмулированной инкапсуляцией представления, такой запрос сможет идентифицировать \(и анимировать\) DOM-элементы на любой глубине дерева.

С другой стороны, `ViewEncapsulation.ShadowDom` и `ViewEncapsulation.ExperimentalIsolatedShadowDom` изменяют структуру DOM компонента, «скрывая» DOM-элементы внутри элементов [`ShadowRoot`](https://developer.mozilla.org/docs/Web/API/ShadowRoot). Такие манипуляции с DOM мешают корректной работе некоторых реализаций анимаций, поскольку они полагаются на простые структуры DOM и не учитывают элементы `ShadowRoot`. Поэтому рекомендуется избегать применения анимаций к представлениям, включающим компоненты с инкапсуляцией представления ShadowDom.

## Итог по последовательностям анимаций {#animation-sequence-summary}

Функции Angular для анимации нескольких элементов начинают с `query()` для поиска внутренних элементов; например, сбора всех изображений внутри `<div>`.
Остальные функции — `stagger()`, [`group()`](api/animations/group) и `sequence()` — применяют каскады или позволяют управлять применением нескольких шагов анимации.
