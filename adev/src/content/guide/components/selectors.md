# Селекторы компонентов {#component-selectors}

TIP: В этом руководстве предполагается, что вы уже ознакомились с [Руководством по основам](essentials). Прочитайте его в первую очередь, если вы новичок в Angular.

Каждый компонент определяет [CSS-селектор](https://developer.mozilla.org/docs/Web/CSS/CSS_selectors), который определяет, как компонент используется:

```angular-ts {highlight: [2]}
@Component({
  selector: 'profile-photo',
  ...
})
export class ProfilePhoto { }
```

Вы используете компонент, создавая соответствующий HTML-элемент в шаблонах _других_ компонентов:

```angular-ts {highlight: [3]}
@Component({
  template: `
    <profile-photo />
    <button>Upload a new profile photo</button>`,
  ...,
})
export class UserProfile { }
```

**Angular сопоставляет селекторы статически во время компиляции**. Изменение DOM во время выполнения — как через привязки Angular, так и через DOM API — не влияет на отображаемые компоненты.

**Один элемент может соответствовать только одному селектору компонента.** Если несколько селекторов компонентов совпадают с одним элементом, Angular сообщит об ошибке.

**Селекторы компонентов чувствительны к регистру.**

## Типы селекторов {#types-of-selectors}

Angular поддерживает ограниченное подмножество [базовых типов CSS-селекторов](https://developer.mozilla.org/docs/Web/CSS/CSS_Selectors) в селекторах компонентов:

| **Тип селектора**       | **Описание**                                                                                                           | **Примеры**                   |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| Селектор типа           | Соответствует элементам на основе их имени HTML-тега или имени узла.                                                   | `profile-photo`               |
| Селектор атрибута       | Соответствует элементам на основе наличия HTML-атрибута и, при необходимости, его точного значения.                     | `[dropzone]` `[type="reset"]` |
| Селектор класса         | Соответствует элементам на основе наличия CSS-класса.                                                                  | `.menu-item`                  |

Для значений атрибутов Angular поддерживает сопоставление точного значения атрибута с помощью оператора равенства (`=`). Angular не поддерживает другие операторы сравнения значений атрибутов.

Селекторы компонентов Angular не поддерживают комбинаторы, включая [комбинатор потомков](https://developer.mozilla.org/docs/Web/CSS/Descendant_combinator) или [дочерний комбинатор](https://developer.mozilla.org/docs/Web/CSS/Child_combinator).

Селекторы компонентов Angular не поддерживают указание [пространств имён](https://developer.mozilla.org/docs/Web/SVG/Namespaces_Crash_Course).

### Псевдокласс `:not` {#the-not-pseudo-class}

Angular поддерживает [псевдокласс `:not`](https://developer.mozilla.org/docs/Web/CSS/:not). Вы можете добавить этот псевдокласс к любому другому селектору, чтобы сузить множество элементов, которым соответствует селектор компонента. Например, вы можете определить селектор атрибута `[dropzone]` и предотвратить сопоставление с элементами `textarea`:

```angular-ts {highlight: [2]}
@Component({
  selector: '[dropzone]:not(textarea)',
  ...
})
export class DropZone { }
```

Angular не поддерживает никаких других псевдоклассов или псевдоэлементов в селекторах компонентов.

### Комбинирование селекторов {#combining-selectors}

Вы можете комбинировать несколько селекторов, объединяя их. Например, можно сопоставить элементы `<button>`, у которых указан `type="reset"`:

```angular-ts {highlight: [2]}
@Component({
  selector: 'button[type="reset"]',
  ...
})
export class ResetButton { }
```

Вы также можете определить несколько селекторов через запятую:

```angular-ts {highlight: [2]}
@Component({
  selector: 'drop-zone, [dropzone]',
  ...
})
export class DropZone { }
```

Angular создает компонент для каждого элемента, который соответствует _любому_ из селекторов в списке.

## Выбор селектора {#choosing-a-selector}

Подавляющее большинство компонентов должны использовать имя пользовательского элемента в качестве селектора. Все имена пользовательских элементов должны содержать дефис, как описано в [спецификации HTML](https://html.spec.whatwg.org/multipage/custom-elements.html#valid-custom-element-name). По умолчанию Angular сообщает об ошибке, если встречает пользовательский тег, который не соответствует ни одному доступному компоненту, предотвращая ошибки из-за опечаток в именах компонентов.

Подробнее об использовании [нативных пользовательских элементов](https://developer.mozilla.org/docs/Web/Web_Components) в шаблонах Angular см. в разделе [Расширенная конфигурация компонентов](guide/components/advanced-configuration).

### Префиксы селекторов {#selector-prefixes}

Команда Angular рекомендует использовать короткий, единообразный префикс для всех пользовательских компонентов, определённых в вашем проекте. Например, если бы вы создавали YouTube на Angular, вы могли бы использовать префикс `yt-` для компонентов вроде `yt-menu`, `yt-player` и т.д. Такое именование позволяет сразу понять, откуда пришёл тот или иной компонент. По умолчанию Angular CLI использует префикс `app-`.

IMPORTANT: Angular использует префикс `ng` для собственных API фреймворка. Никогда не используйте `ng` в качестве префикса селектора для ваших пользовательских компонентов.

### Когда использовать селектор атрибута {#when-to-use-an-attribute-selector}

Следует рассмотреть использование селектора атрибута, когда вы хотите создать компонент на основе стандартного нативного элемента. Например, если вы хотите создать компонент кнопки, вы можете воспользоваться стандартным элементом `<button>`, применив селектор атрибута:

```angular-ts {highlight: [2]}
@Component({
  selector: 'button[yt-upload]',
   ...
})
export class YouTubeUploadButton { }
```

Такой подход позволяет пользователям компонента напрямую использовать все стандартные API элемента без дополнительных усилий. Это особенно важно для ARIA-атрибутов, таких как `aria-label`.

Angular не сообщает об ошибках, когда встречает пользовательские атрибуты, которые не соответствуют доступному компоненту. При использовании компонентов с селекторами атрибутов разработчики могут забыть импортировать компонент или его NgModule, что приведёт к тому, что компонент не будет отображаться.
Подробнее см. в разделе [Импорт и использование компонентов](guide/components#imports-in-the-component-decorator).

Компоненты с селекторами атрибутов должны использовать атрибуты в нижнем регистре с дефисами. Вы можете следовать тем же рекомендациям по использованию префиксов, описанным выше.
