# Утилиты тестирования Zone.js {#zonejs-testing-utilities}

В этом руководстве описаны утилиты тестирования, используемые прежде всего для управления и контроля асинхронных задач в модульных тестах. По сути это специфичные для Zone.js утилиты mock-часов, особенно актуальные для управления потоком асинхронных операций в тестах.

Общие утилиты тестирования Angular, включая `TestBed` и `ComponentFixture`, см. в руководстве [API утилит тестирования](guide/testing/utility-apis).

Вот краткое описание специфичных для Zone.js функций:

| Функция                | Подробности                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| :--------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `waitForAsync`         | Отслеживает асинхронные задачи и завершает тесты только после того, как в тестовой зоне не останется ни микро-, ни макрозадач.                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `fakeAsync`            | Запускает тело теста \(`it`\) в специальной _тестовой зоне fakeAsync_, обеспечивая линейный стиль написания кода.                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| `tick`                 | Имитирует течение времени и завершение ожидающих асинхронных операций, очищая очереди _таймеров_ и _микрозадач_ внутри _тестовой зоны fakeAsync_. Любознательному читателю может быть интересна эта обширная статья блога: [«_Tasks, microtasks, queues and schedules_»](https://jakearchibald.com/2015/tasks-microtasks-queues-and-schedules). Принимает необязательный аргумент, продвигающий виртуальные часы вперёд на указанное количество миллисекунд, очищая асинхронные операции, запланированные в этом временном промежутке. |
| `discardPeriodicTasks` | Удаляет все периодические задачи (например, `setInterval`), созданные внутри зоны `fakeAsync`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `flushMicrotasks`      | Когда тест `fakeAsync()` завершается с ожидающими _микрозадачами_, такими как неразрешённые промисы, тест завершается ошибкой с понятным сообщением. <br /> В общем случае тест должен ожидать завершения микрозадач. Если ожидаются отложенные микрозадачи, вызовите `flushMicrotasks` для очистки очереди _микрозадач_ и предотвращения ошибки.                                                                                                                                                                                                                    |
