# Утилиты тестирования Zone.js {#zonejs-testing-utilities}

Это руководство описывает утилиты тестирования, используемые преимущественно для управления и контроля асинхронных задач в юнит-тестах. По существу это специфичные для Zone.js утилиты поддельных часов, особенно актуальные для управления потоком асинхронных операций в тестах.

Информацию об общих утилитах тестирования Angular, включая `TestBed` и `ComponentFixture`, смотрите в руководстве [API утилит тестирования](guide/testing/utility-apis).

Краткое описание функций, специфичных для Zone.js:

| Функция                | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| :--------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `waitForAsync`         | Отслеживает асинхронные задачи и завершает тесты только после того, как в тестовой зоне не останется ни микро-, ни макрозадач.                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `fakeAsync`            | Выполняет тело теста \(`it`\) в специальной _тестовой зоне fakeAsync_, что обеспечивает линейный стиль управления потоком.                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `tick`                 | Имитирует течение времени и завершение ожидающих асинхронных операций путём очистки очередей _таймеров_ и _микрозадач_ в _тестовой зоне fakeAsync_. Любознательным читателям может быть интересна статья [«_Tasks, microtasks, queues and schedules_»](https://jakearchibald.com/2015/tasks-microtasks-queues-and-schedules). Принимает необязательный аргумент, сдвигающий виртуальные часы вперёд на указанное количество миллисекунд, выполняя асинхронные операции, запланированные в этом временном интервале. |
| `discardPeriodicTasks` | Удаляет все периодические задачи (например, `setInterval`), созданные внутри зоны `fakeAsync`.                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| `flushMicrotasks`      | Когда тест `fakeAsync()` завершается с ожидающими _микрозадачами_, такими как нерешённые промисы, тест завершается с понятным сообщением об ошибке. <br /> В общем случае тест должен дождаться завершения микрозадач. Если ожидающие микрозадачи ожидаемы, вызовите `flushMicrotasks` для очистки очереди _микрозадач_ и предотвращения ошибки.                                                                                                                                                                                                    |
