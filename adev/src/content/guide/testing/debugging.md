# Отладка тестов {#debugging-tests}

Если тесты работают не так, как вы ожидаете, их можно отлаживать как в стандартной среде Node.js, так и в реальном браузере.

## Отладка в Node.js {#debugging-in-nodejs}

Отладка в стандартной среде Node.js зачастую является самым быстрым способом диагностики проблем, не связанных с API или рендерингом, специфичными для браузера.

1.  Запустите команду `ng test` с флагом `--debug`:
    ```shell
    ng test --debug
    ```
2.  Средство запуска тестов запустится в режиме отладки и будет ожидать подключения отладчика.
3.  Теперь можно подключить предпочтительный отладчик. Например, можно использовать встроенный отладчик Node.js в VS Code или Chrome DevTools для Node.js.

## Отладка в браузере {#debugging-in-a-browser}

Так же, как и при запуске сессии отладки в Node, можно использовать `ng test` с флагом `--debug` вместе с Vitest и [браузерным режимом](/guide/testing/migrating-to-vitest#5-configure-browser-mode-optional).

Средство запуска тестов запустится в режиме отладки и будет ожидать, пока вы откроете инструменты разработчика браузера для отладки тестов.
