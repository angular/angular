# Обзор Angular Elements

_Angular elements_ — это Angular-компоненты, упакованные как _пользовательские элементы_ \(также называемые Web Components\), веб-стандарт для определения новых HTML-элементов независимым от фреймворка способом.

[Пользовательские элементы](https://developer.mozilla.org/docs/Web/Web_Components/Using_custom_elements) — это функция веб-платформы, доступная во всех браузерах, поддерживаемых Angular.
Пользовательский элемент расширяет HTML, позволяя определить тег, содержимое которого создаётся и управляется кодом JavaScript.
Браузер поддерживает `CustomElementRegistry` определённых пользовательских элементов, которая отображает инстанцируемый JavaScript-класс на HTML-тег.

Пакет `@angular/elements` экспортирует API `createCustomElement()`, который обеспечивает мост от интерфейса компонентов Angular и функциональности обнаружения изменений к встроенному DOM API.

Преобразование компонента в пользовательский элемент делает всю необходимую инфраструктуру Angular доступной для браузера.
Создание пользовательского элемента просто и понятно, и автоматически связывает определённое в компоненте представление с обнаружением изменений и привязкой данных, отображая функциональность Angular на соответствующие встроенные эквиваленты HTML.

## Использование пользовательских элементов {#using-custom-elements}

Пользовательские элементы запускают себя сами — они запускаются при добавлении в DOM и уничтожаются при удалении из DOM.
После добавления пользовательского элемента в DOM для любой страницы он выглядит и ведёт себя как любой другой HTML-элемент и не требует специальных знаний терминологии или соглашений Angular.

Чтобы добавить пакет `@angular/elements` в ваше рабочее пространство, выполните следующую команду:

### Как это работает {#how-it-works}

Функция `createCustomElement()` преобразует компонент в класс, который может быть зарегистрирован в браузере как пользовательский элемент.
После регистрации настроенного класса в реестре пользовательских элементов браузера используйте новый элемент так же, как встроенный HTML-элемент, в контенте, добавляемом непосредственно в DOM:

```html
<my-popup message="Use Angular!"></my-popup>
```

Когда пользовательский элемент размещается на странице, браузер создаёт экземпляр зарегистрированного класса и добавляет его в DOM.
Контент предоставляется шаблоном компонента, использующим синтаксис шаблонов Angular, и рендерится с использованием данных компонента и DOM.
Input-свойства компонента соответствуют входным атрибутам элемента.

## Преобразование компонентов в пользовательские элементы {#transforming-components-to-custom-elements}

Angular предоставляет функцию `createCustomElement()` для преобразования Angular-компонента вместе с его зависимостями в пользовательский элемент.

Процесс преобразования реализует интерфейс `NgElementConstructor` и создаёт
класс-конструктор, настроенный для создания самозагружающегося экземпляра вашего компонента.

Используйте встроенную функцию браузера [`customElements.define()`](https://developer.mozilla.org/docs/Web/API/CustomElementRegistry/define) для регистрации настроенного конструктора и связанного с ним тега пользовательского элемента в [`CustomElementRegistry`](https://developer.mozilla.org/docs/Web/API/CustomElementRegistry) браузера.
Когда браузер встречает тег зарегистрированного элемента, он использует конструктор для создания экземпляра пользовательского элемента.

IMPORTANT: Избегайте использования селектора компонента в качестве имени тега пользовательского элемента.
Это может привести к неожиданному поведению из-за создания Angular двух экземпляров компонента для одного DOM-элемента:
одного обычного Angular-компонента и второго — с использованием пользовательского элемента.

### Отображение {#mapping}

Пользовательский элемент _содержит_ Angular-компонент, обеспечивая мост между данными и логикой, определёнными в компоненте, и стандартными DOM API.
Свойства и логика компонента напрямую отображаются на HTML-атрибуты и систему событий браузера.

- API создания анализирует компонент, ища input-свойства, и определяет соответствующие атрибуты для пользовательского элемента.
  Он преобразует имена свойств, чтобы сделать их совместимыми с пользовательскими элементами, которые не распознают различия регистра.
  Результирующие имена атрибутов используют строчные буквы с разделением дефисом.
  Например, для компонента с `inputProp = input({alias: 'myInputProp'})` соответствующий пользовательский элемент определяет атрибут `my-input-prop`.

- Выходные данные компонента передаются как HTML [Custom Events](https://developer.mozilla.org/docs/Web/API/CustomEvent), при этом имя пользовательского события совпадает с именем выходного свойства.
  Например, для компонента с `valueChanged = output()` соответствующий пользовательский элемент генерирует события с именем "valueChanged", а отправленные данные хранятся в свойстве `detail` события.
  Если указан псевдоним, используется его значение; например, `clicks = output<string>({alias: 'myClick'});` приводит к генерации событий с именем "myClick".

Дополнительные сведения см. в документации Web Component по [созданию пользовательских событий](https://developer.mozilla.org/en-US/docs/Web/API/Document_Object_Model/Events#creating_custom_events).

## Пример: сервис Popup {#example-a-popup-service}

Прежде чем, когда нужно было добавить компонент в приложение во время выполнения, требовалось определить _динамический компонент_, а затем загрузить его, прикрепить к элементу в DOM и подключить все зависимости, обнаружение изменений и обработку событий.

Использование Angular-пользовательского элемента упрощает и делает прозрачным этот процесс, предоставляя всю инфраструктуру и фреймворк автоматически — всё, что нужно сделать, — определить нужный вид обработки событий.
\(Вам по-прежнему нужно исключить компонент из компиляции, если вы не собираетесь использовать его в своём приложении.\)

В следующем примере приложения Popup Service определяется компонент, который можно загружать динамически или преобразовать в пользовательский элемент.

| Файлы              | Подробности                                                                                                                                                                                                             |
| :----------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `popup.ts`         | Определяет простой всплывающий элемент, отображающий входное сообщение с некоторой анимацией и стилями.                                                                                                                 |
| `popup.service.ts` | Создаёт внедряемый сервис, предоставляющий два разных способа вызова `Popup`: как динамический компонент или как пользовательский элемент. Обратите внимание, насколько больше настроек требует метод динамической загрузки. |
| `app.ts`           | Определяет корневой компонент приложения, использующий `PopupService` для добавления всплывающего окна в DOM во время выполнения. При запуске приложения конструктор корневого компонента преобразует `Popup` в пользовательский элемент. |

## Типизация для пользовательских элементов {#typings-for-custom-elements}

Универсальные DOM API, такие как `document.createElement()` или `document.querySelector()`, возвращают тип элемента, подходящий для указанных аргументов.
Например, вызов `document.createElement('a')` возвращает `HTMLAnchorElement`, который TypeScript знает как имеющий свойство `href`.
Аналогично, `document.createElement('div')` возвращает `HTMLDivElement`, который TypeScript знает как не имеющий свойства `href`.

При вызове с неизвестными элементами, такими как имя пользовательского элемента \(`popup-element` в нашем примере\), методы возвращают универсальный тип, такой как `HTMLElement`, поскольку TypeScript не может определить правильный тип возвращаемого элемента.

Пользовательские элементы, созданные с помощью Angular, расширяют `NgElement` \(который в свою очередь расширяет `HTMLElement`\).
Кроме того, у этих пользовательских элементов будет свойство для каждого input соответствующего компонента.
Например, наш `popup-element` имеет свойство `message` типа `string`.

Есть несколько вариантов получения правильных типов для пользовательских элементов.
Предположим, вы создаёте пользовательский элемент `my-dialog` на основе следующего компонента:

```ts
@Component(/* ... */)
class MyDialog {
  content = input('');
}
```

Самый простой способ получить точную типизацию — привести возвращаемое значение соответствующих DOM-методов к правильному типу.
Для этого используйте типы `NgElement` и `WithProperties` \(оба экспортируются из `@angular/elements`\):

```ts
const aDialog = document.createElement('my-dialog') as NgElement &
  WithProperties<{content: string}>;
aDialog.content = 'Hello, world!';
aDialog.content = 123; // <-- ERROR: TypeScript knows this should be a string.
aDialog.body = 'News'; // <-- ERROR: TypeScript knows there is no `body` property on `aDialog`.
```

Это хороший способ быстро получить функции TypeScript, такие как проверка типов и поддержка автодополнения, для вашего пользовательского элемента.
Но это может стать обременительным, если это нужно в нескольких местах, поскольку придётся приводить тип возвращаемого значения при каждом использовании.

Альтернативный способ, требующий определения типа каждого пользовательского элемента только один раз, — дополнение `HTMLElementTagNameMap`, которое TypeScript использует для определения типа возвращаемого элемента на основе его имени тега \(для DOM-методов, таких как `document.createElement()`, `document.querySelector()` и т. д.\):

```ts

declare global {
  interface HTMLElementTagNameMap {
    'my-dialog': NgElement & WithProperties<{content: string}>;
    'my-other-element': NgElement & WithProperties<{foo: 'bar'}>;
    …
  }
}

```

Теперь TypeScript может определить правильный тип так же, как и для встроенных элементов:

```ts
document.createElement('div'); //--> HTMLDivElement (built-in element)
document.querySelector('foo'); //--> Element        (unknown element)
document.createElement('my-dialog'); //--> NgElement & WithProperties<{content: string}> (custom element)
document.querySelector('my-other-element'); //--> NgElement & WithProperties<{foo: 'bar'}>      (custom element)
```

## Ограничения {#limitations}

При уничтожении и повторном присоединении пользовательских элементов, созданных с помощью `@angular/elements`, следует проявлять осторожность из-за проблем с обратным вызовом [disconnect()](https://github.com/angular/angular/issues/38778). Случаи, в которых вы можете столкнуться с этой проблемой:

- Рендеринг компонента в `ng-if` или `ng-repeat` в `AngularJS`
- Ручное отсоединение и повторное присоединение элемента к DOM
