Каждый компонент Angular имеет **шаблон**, определяющий [DOM](https://developer.mozilla.org/en-US/docs/Web/API/Document_Object_Model), который компонент рендерит на странице. Благодаря шаблонам Angular автоматически обновляет страницу при изменении данных.

Шаблоны обычно находятся либо в свойстве `template` файла `*.ts`, либо в файле `*.html`. Чтобы узнать больше, ознакомьтесь с [подробным руководством по компонентам](/guide/components).

## Как работают шаблоны {#how-do-templates-work}

Шаблоны основаны на синтаксисе [HTML](https://developer.mozilla.org/en-US/docs/Web/HTML) с дополнительными возможностями, такими как встроенные функции шаблонов, привязка данных, обработка событий, переменные и многое другое.

Angular компилирует шаблоны в JavaScript для формирования внутреннего представления вашего приложения. Одним из преимуществ этого подхода являются встроенные оптимизации рендеринга, которые Angular автоматически применяет к вашему приложению.

### Отличия от стандартного HTML {#differences-from-standard-html}

Некоторые отличия между шаблонами и стандартным синтаксисом HTML:

- Комментарии в исходном коде шаблона не включаются в отрендеренный результат
- Элементы компонентов и директив могут быть самозакрывающимися (например, `<UserProfile />`)
- Атрибуты с определенными символами (например, `[]`, `()` и т.д.) имеют специальное значение в Angular. См. [документацию по привязке](guide/templates/binding) и [документацию по обработчикам событий](guide/templates/event-listeners) для получения дополнительной информации.
- Символ `@` имеет специальное значение в Angular для добавления динамического поведения, например [управления потоком](guide/templates/control-flow), в шаблоны. Вы можете включить буквальный символ `@`, экранировав его как HTML-сущность (`&commat;` или `&#64;`).
- Angular игнорирует и сворачивает ненужные пробельные символы. Подробнее см. [Пробелы в шаблонах](guide/templates/whitespace).
- Angular может добавлять узлы комментариев на страницу в качестве заполнителей для динамического контента, но разработчики могут их игнорировать.

Кроме того, хотя большая часть HTML-синтаксиса является допустимым синтаксисом шаблонов, Angular не поддерживает элемент `<script>` в шаблонах. Для получения дополнительной информации см. страницу [Безопасность](best-practices/security).
