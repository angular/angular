# Настройка Angular CLI MCP Server {#angular-cli-mcp-server-setup}

Angular CLI включает экспериментальный [сервер Model Context Protocol (MCP)](https://modelcontextprotocol.io/), позволяющий ИИ-ассистентам в вашей среде разработки взаимодействовать с Angular CLI. Мы добавили поддержку генерации кода с помощью CLI, добавления пакетов и многого другого.

## Доступные инструменты {#available-tools}

MCP Server Angular CLI предоставляет несколько инструментов для помощи в рабочем процессе разработки. По умолчанию включены следующие инструменты:

| Название                    | Описание                                                                                                                                                                                                     | `local-only` | `read-only` |
| :-------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------: | :---------: |
| `ai_tutor`                  | Запускает интерактивный Angular-тьютор на основе ИИ. Рекомендуется запускать из нового Angular-проекта версии v20 или выше. [Подробнее](ai/ai-tutor).                                                       |      ✅      |     ✅      |
| `find_examples`             | Находит авторитетные примеры кода из курируемой базы данных официальных примеров лучших практик, фокусируясь на **современных, новых и недавно обновлённых** функциях Angular.                              |      ✅      |     ✅      |
| `get_best_practices`        | Извлекает руководство Angular Best Practices. Это руководство необходимо для обеспечения соответствия всего кода современным стандартам, включая автономные компоненты, типизированные формы и современный поток управления. |      ✅      |     ✅      |
| `list_projects`             | Выводит имена всех приложений и библиотек, определённых в Angular workspace. Читает файл конфигурации `angular.json` для идентификации проектов.                                                            |      ✅      |     ✅      |
| `onpush_zoneless_migration` | Анализирует код Angular и предоставляет пошаговый итерационный план для его миграции на обнаружение изменений `OnPush` — предварительное условие для zoneless-приложения.                                    |      ✅      |     ✅      |
| `search_documentation`      | Ищет официальную документацию Angular на <https://angular.dev>. Этот инструмент следует использовать для ответа на любые вопросы об Angular, такие как API, учебные пособия и лучшие практики.             |      ❌      |     ✅      |

### Экспериментальные инструменты {#experimental-tools}

Некоторые инструменты предоставляются в экспериментальном/предварительном статусе, поскольку они новые или ещё не полностью протестированы. Включайте их по отдельности с помощью опции [`--experimental-tool`](#command-options) и используйте с осторожностью.

| Название                   | Описание                                                                                                                                                                                                                                                                                     | `local-only` | `read-only` |
| :------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------: | :---------: |
| `build`                    | Выполняет единовременную сборку без отслеживания изменений с помощью `ng build`.                                                                                                                                                                                                             |      ✅      |     ❌      |
| `devserver.start`          | Асинхронно запускает сервер разработки, отслеживающий изменения в workspace, аналогично запуску `ng serve`. Поскольку это асинхронно, возвращается немедленно. Для управления результирующим сервером используйте инструменты `devserver.stop` и `devserver.wait_for_build`. |      ✅      |     ✅      |
| `devserver.stop`           | Останавливает сервер разработки, запущенный `devserver.start`.                                                                                                                                                                                                                               |      ✅      |     ✅      |
| `devserver.wait_for_build` | Возвращает выходные логи последней сборки в работающем сервере разработки, запущенном `devserver.start`. Если сборка в данный момент выполняется, сначала ожидает её завершения, затем возвращает логи.                                                                                     |      ✅      |     ✅      |
| `e2e`                      | Выполняет end-to-end тесты, настроенные в проекте.                                                                                                                                                                                                                                           |      ✅      |     ✅      |
| `modernize`                | Выполняет миграции кода и предоставляет дополнительные инструкции по модернизации кода Angular в соответствии с последними лучшими практиками и синтаксисом. [Подробнее](https://angular.dev/reference/migrations)                                                                           |      ✅      |     ❌      |
| `test`                     | Запускает модульные тесты проекта.                                                                                                                                                                                                                                                            |      ✅      |     ✅      |

## Начало работы {#get-started}

Для начала работы выполните следующую команду в терминале:

```bash
ng mcp
```

При запуске из интерактивного терминала эта команда отображает инструкции по настройке хост-среды для использования MCP server. В следующих разделах приведены примеры конфигураций для нескольких популярных редакторов и инструментов.

### Cursor {#cursor}

Создайте файл `.cursor/mcp.json` в корне проекта и добавьте следующую конфигурацию. Также можно настроить глобально в `~/.cursor/mcp.json`.

```json
{
  "mcpServers": {
    "angular-cli": {
      "command": "npx",
      "args": ["-y", "@angular/cli", "mcp"]
    }
  }
}
```

### Firebase Studio {#firebase-studio}

Создайте файл `.idx/mcp.json` в корне проекта и добавьте следующую конфигурацию:

```json
{
  "mcpServers": {
    "angular-cli": {
      "command": "npx",
      "args": ["-y", "@angular/cli", "mcp"]
    }
  }
}
```

### Gemini CLI {#gemini-cli}

Создайте файл `.gemini/settings.json` в корне проекта и добавьте следующую конфигурацию:

```json
{
  "mcpServers": {
    "angular-cli": {
      "command": "npx",
      "args": ["-y", "@angular/cli", "mcp"]
    }
  }
}
```

### JetBrains IDE {#jetbrains-ides}

В JetBrains IDE (например, IntelliJ IDEA или WebStorm) после установки плагина JetBrains AI Assistant перейдите в `Settings | Tools | AI Assistant | Model Context Protocol (MCP)`. Добавьте новый сервер (`+`) и выберите `As JSON`. Затем вставьте следующую конфигурацию:

```json
{
  "mcpServers": {
    "angular-cli": {
      "command": "npx",
      "args": ["-y", "@angular/cli", "mcp"]
    }
  }
}
```

Для получения самых актуальных инструкций по настройке MCP-серверов обратитесь к документации JetBrains: [Connect to an MCP server](https://www.jetbrains.com/help/ai-assistant/mcp.html#connect-to-an-mcp-server).

### VS Code {#vs-code}

В корне проекта создайте файл `.vscode/mcp.json` и добавьте следующую конфигурацию. Обратите внимание на использование свойства `servers`.

```json
{
  "servers": {
    "angular-cli": {
      "command": "npx",
      "args": ["-y", "@angular/cli", "mcp"]
    }
  }
}
```

### Другие IDE {#other-ides}

Для других IDE обратитесь к документации IDE для определения правильного расположения файла конфигурации MCP (часто `mcp.json`). Конфигурация должна содержать следующий фрагмент.

```json
{
  "mcpServers": {
    "angular-cli": {
      "command": "npx",
      "args": ["-y", "@angular/cli", "mcp"]
    }
  }
}
```

## Параметры команды {#command-options}

Команда `mcp` может быть настроена со следующими параметрами, передаваемыми в качестве аргументов в конфигурации MCP вашей IDE:

| Параметр                      | Тип       | Описание                                                                                                                                                                      | По умолчанию |
| :---------------------------- | :-------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------- |
| `--read-only`                 | `boolean` | Регистрировать только инструменты, не вносящие изменений в проект. Ваш редактор или агент кодирования всё равно может выполнять правки.                                       | `false`      |
| `--local-only`                | `boolean` | Регистрировать только инструменты, не требующие подключения к интернету. Ваш редактор или агент кодирования всё равно может отправлять данные по сети.                        | `false`      |
| `--experimental-tool`<br>`-E` | `string`  | Включить [экспериментальный инструмент](#experimental-tools). Разделяйте несколько параметров пробелами, например `-E tool_a tool_b`. Включите все инструменты `devserver.x` с помощью `-E devserver`. |              |

Например, для запуска сервера в режиме только для чтения в VS Code нужно обновить `mcp.json` следующим образом:

```json
{
  "servers": {
    "angular-cli": {
      "command": "npx",
      "args": ["-y", "@angular/cli", "mcp", "--read-only"]
    }
  }
}
```
